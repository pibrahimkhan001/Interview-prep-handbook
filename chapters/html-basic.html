Question 1

What does HTML stand for?

HTML stands for HyperText Markup Language. It’s the standard language used to create and structure the content of web pages.

  • HyperText — text that links to other text (the hyperlinks that connect pages across the web).
  • Markup — tags wrap content to describe what it is (a heading, a paragraph, a list, an image).
  • Language — a defined set of rules browsers understand and render.
<h1>Hello, world!</h1>
<p>This is a paragraph.</p>

HTML isn’t a programming language — it has no variables, conditions, or loops. It’s a markup language: it describes structure. CSS handles appearance and JavaScript handles behavior. The current standard is HTML Living Standard, maintained by the WHATWG.

Question 2

What is the purpose of the <!DOCTYPE html> declaration?

The <!DOCTYPE html> declaration tells the browser: “render this page in standards mode using the HTML5 specification.” It must be the very first line of every HTML document, before any other tag.

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>My Page</title>
  </head>
  <body>...</body>
</html>

Without it, browsers fall back to quirks mode, which emulates 1990s bugs for old-site compatibility — causing unpredictable layout and broken CSS. Omitting <!DOCTYPE> is almost always a mistake.

💡
In HTML5 this declaration is short and case-insensitive. Older versions (XHTML, HTML4) had longer, stricter forms — but in 2026 you’ll only ever need <!DOCTYPE html>.
Question 3

How do you create a hyperlink in HTML?

A hyperlink is created with the anchor tag <a> and its href attribute (short for hypertext reference). The text between the opening and closing tags becomes clickable.

<a href="https://example.com">Visit Example</a>
<a href="/about">About page (same site)</a>
<a href="#section-2">Jump to section 2</a>

href accepts three common forms:

  • Absolute URL — full address including protocol (https://...).
  • Relative URL — path on the same site (/about or ../help.html).
  • Fragment — jump to an element with a matching id on the current page (#top).

For links that open in a new tab, add target="_blank". When doing so, also add rel="noopener" to prevent the new page from tampering with the original — a small security safeguard.

Question 4

What are the different types of lists in HTML?

HTML has three types of lists, each serving a different purpose:

  • Unordered list (<ul>) — items have no sequence; rendered with bullets. Use for groupings where order doesn’t matter.
  • Ordered list (<ol>) — items follow a sequence; rendered with numbers. Use for steps, rankings, or anything order-dependent.
  • Description list (<dl>) — pairs of terms and their descriptions. Use for glossaries, metadata, or FAQs.
<ul>
  <li>Apple</li>
  <li>Banana</li>
</ul>

<ol>
  <li>Preheat oven</li>
  <li>Mix ingredients</li>
</ol>

<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
</dl>

Picking the right list type matters for accessibility — screen readers announce “numbered list of 5 items” vs “list of 5 items,” which gives visually impaired users helpful context about the content.

Question 5

How do you create an ordered list in HTML?

An ordered list is created with the <ol> tag, and each item is wrapped in an <li> (list item) tag. The browser numbers them automatically.

<ol>
  <li>Wake up</li>
  <li>Brush teeth</li>
  <li>Drink coffee</li>
</ol>

By default items number as 1, 2, 3. You can customize with attributes:

  • type="A" — uppercase letters (A, B, C).
  • type="a" — lowercase letters (a, b, c).
  • type="I" — uppercase Roman (I, II, III).
  • start="5" — begin counting from 5.
  • reversed — count down instead of up.
<ol type="A" start="3">
  <li>First item shown (labeled C)</li>
</ol>

Use an ordered list whenever sequence matters — recipes, rankings, instructions. If you only want styled bullets or numbers without semantic order, CSS is the right tool.

Question 6

How do you create an unordered list in HTML?

An unordered list uses the <ul> tag with <li> (list item) children. Browsers render each item with a bullet point; the order of items carries no meaning.

<ul>
  <li>Red</li>
  <li>Green</li>
  <li>Blue</li>
</ul>

The default bullet marker can be changed with CSS using the list-style-type property:

ul { list-style-type: square; }  /* or circle, disc, none */

Unordered lists are one of the most commonly used HTML elements — they’re the right choice for navigation menus, feature lists, tags, product bullet points, and anywhere items belong together but have no inherent order. Screen readers announce the list size, which helps users navigate quickly.

Question 7

How do you add an image to a webpage in HTML?

Images are added with the self-closing <img> tag. It requires two attributes: src (the image path) and alt (a text description for accessibility and fallback).

<img src="photo.jpg" alt="A sunset over the ocean">
<img src="https://cdn.example.com/logo.png" alt="Company logo">

Common attributes:

  • src — the file path, URL, or data URI.
  • alt — text shown if the image fails to load; read aloud by screen readers.
  • width / height — reserve layout space so the page doesn’t jump when images load.
  • loading="lazy" — defer loading until the image is near the viewport; improves page speed.
<img src="hero.jpg" alt="" width="800" height="450" loading="lazy">

Always set width and height when you know them — it prevents layout shift (content jumping as images arrive), which hurts both user experience and Core Web Vitals scores.

Question 8

What is the purpose of the <alt> attribute in an <img> tag?

The alt attribute gives a text description of the image. It serves several important purposes:

  • Accessibility — screen readers read alt text aloud so blind and low-vision users know what the image shows.
  • Fallback — if the image fails to load (broken link, slow connection), the browser displays the alt text in its place.
  • SEO — search engines use alt text to understand image content and index it appropriately.
<img src="puppy.jpg" alt="Golden retriever puppy playing in grass">

<!-- Decorative image — use empty alt so screen readers skip it -->
<img src="divider.png" alt="">

Good alt text is concise and descriptive. Describe the content and function, not the appearance (“Sign up” button, not “Green button”). Skip phrases like “Image of” — screen readers already announce it’s an image. For purely decorative images with no informational value, use alt="" so assistive tech ignores them.

Question 9

How do you create a table in HTML?

A table is created with the <table> tag, containing rows (<tr>) which hold either header cells (<th>) or data cells (<td>).

<table>
  <thead>
    <tr>
      <th>Name</th>
      <th>Age</th>
      <th>City</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Alice</td>
      <td>30</td>
      <td>Paris</td>
    </tr>
    <tr>
      <td>Bob</td>
      <td>25</td>
      <td>Tokyo</td>
    </tr>
  </tbody>
</table>

Structural elements (<thead>, <tbody>, <tfoot>) group rows semantically and help screen readers announce “row 2 of 50” correctly. Always use <th> for headers — not a styled <td> — since it carries meaning for assistive technologies.

Tables are for tabular data only — not for page layout. Use CSS Grid or Flexbox for layout; they’re more flexible and semantic.

Question 10

How do you create a form in HTML?

A form is created with the <form> tag, which wraps input controls. Two attributes control submission behavior: action (where to send data) and method (how to send it).

<form action="/signup" method="post">
  <label for="email">Email</label>
  <input type="email" id="email" name="email" required>

  <label for="pwd">Password</label>
  <input type="password" id="pwd" name="pwd" required>

  <button type="submit">Sign up</button>
</form>

Key points:

  • action — URL that receives the submission. Omitting it posts to the current page.
  • method="get" — values appear in the URL as query parameters. Good for searches, idempotent.
  • method="post" — values sent in the request body. Use for sign-ups, logins, anything that changes server state.
  • name on each input is required — that’s the key under which its value arrives at the server.

Without the name attribute, an input’s value is not included in the submission — a common beginner gotcha.

Question 11

What is the purpose of the <label> tag in an HTML form?

The <label> tag associates descriptive text with a form control. It makes forms accessible and easier to use.

<!-- Method 1: for/id pairing -->
<label for="user">Username</label>
<input type="text" id="user" name="user">

<!-- Method 2: wrapping the input -->
<label>
  Email
  <input type="email" name="email">
</label>

Benefits of using <label>:

  • Clicking the label focuses the input — a larger, more forgiving click target (especially helpful on mobile).
  • Screen readers announce the label when users focus the input — essential for blind users.
  • Checkboxes and radios become clickable via their label — a big usability win.

Never use placeholder text as a substitute for a label — it disappears when the user types, leaving them without context. Always pair every input with a proper <label>.

Question 12

How do you create a text input field in a form?

A single-line text input is created with <input type="text">. The name attribute is the key the value is submitted under.

<label for="fullname">Full name</label>
<input type="text" id="fullname" name="fullname" placeholder="Jane Doe">

Useful attributes:

  • placeholder — hint text shown when empty (not a substitute for a label).
  • value — initial/pre-filled text.
  • required — browser blocks submission if left empty.
  • maxlength / minlength — character limits.
  • pattern — regex for validation.
  • autocomplete — help browsers suggest values (name, email, off...).
<input type="text" name="zip" pattern="[0-9]{5}" maxlength="5" required>

For specific content, prefer typed inputs: type="email", type="tel", type="url", type="number". Mobile browsers show a matching keyboard layout, and some validation is automatic.

Question 13

How do you create a submit button in a form?

A submit button triggers form submission. Two equivalent ways:

<!-- Preferred: <button> element -->
<button type="submit">Sign up</button>

<!-- Alternative: <input> -->
<input type="submit" value="Sign up">

Inside a <form>, a <button> with no explicit type defaults to type="submit" — so always be explicit to avoid surprises.

  • type="submit" — submits the form (default inside a form).
  • type="reset" — clears all fields to their initial values.
  • type="button" — does nothing by default; used with JavaScript event handlers.

<button> is preferred over <input type="submit"> because it can contain HTML (icons, spans for styling), while <input> only supports plain text via value:

<button type="submit">
  <img src="check.svg" alt=""> <span>Save changes</span>
</button>
Question 14

How do you create a checkbox in a form?

A checkbox is created with <input type="checkbox">. Users can toggle it independently of other checkboxes — any combination can be selected.

<label>
  <input type="checkbox" name="subscribe" value="yes">
  Subscribe to newsletter
</label>

<fieldset>
  <legend>Select interests</legend>
  <label><input type="checkbox" name="interests" value="tech"> Tech</label>
  <label><input type="checkbox" name="interests" value="sports"> Sports</label>
  <label><input type="checkbox" name="interests" value="music" checked> Music</label>
</fieldset>

Key points:

  • checked — pre-selects the checkbox.
  • value — what’s submitted when checked. Defaults to "on" if omitted.
  • Only checked boxes submit data — unchecked boxes send nothing. If you need to know whether the user saw and unchecked a box, use a hidden input or handle it server-side.
  • Use the same name for related checkboxes to submit multiple values for the same field.
Question 15

How do you create a radio button in a form?

A radio button is created with <input type="radio">. Unlike checkboxes, only one radio in a group can be selected at a time. Grouping is done by giving multiple radios the same name.

<fieldset>
  <legend>Preferred contact</legend>

  <label>
    <input type="radio" name="contact" value="email" checked>
    Email
  </label>
  <label>
    <input type="radio" name="contact" value="phone">
    Phone
  </label>
  <label>
    <input type="radio" name="contact" value="sms">
    SMS
  </label>
</fieldset>

Notes:

  • Same name = same group. Different name = independent choices.
  • Different value for each option — that’s what’s submitted.
  • Use checked on the default option; radios without a default can confuse users (nothing pre-selected).
  • Group related radios inside a <fieldset> with a <legend> — screen readers announce the legend as the question being answered.

Rule of thumb: checkboxes for many-of-many; radios for one-of-many.

Question 16

What is the purpose of the <textarea> tag in a form?

The <textarea> tag creates a multi-line text input — essential for comments, messages, descriptions, code snippets, or anything longer than a single line.

<label for="msg">Your message</label>
<textarea id="msg" name="message" rows="5" cols="40" placeholder="Type here..."></textarea>

Unlike <input>, <textarea> has an opening and closing tag; initial content goes between them:

<textarea name="bio">
Initial text
appears here.
</textarea>

Common attributes:

  • rows / cols — visible lines and character width (CSS is usually preferred for sizing).
  • maxlength — character limit.
  • wrap — controls line-wrapping in the submitted value (soft or hard).
  • placeholder and required work the same as on <input>.

Whitespace inside the tag counts as initial text — avoid line breaks between <textarea> and your content if you want it truly empty.

Question 17

How do you create a dropdown list in a form?

A dropdown list is built from two tags working together: <select> wraps the menu, and <option> tags define each choice.

<label for="country">Country</label>
<select id="country" name="country">
  <option value="">-- Choose --</option>
  <option value="us">United States</option>
  <option value="uk">United Kingdom</option>
  <option value="fr" selected>France</option>
</select>

Useful features:

  • selected — pre-selects an option.
  • disabled on an option — visible but not choosable.
  • multiple attribute on <select> — allows picking several options (Ctrl/Cmd-click).
  • <optgroup label="..."> — groups related options with a non-selectable label.
<select name="pet" multiple size="4">
  <optgroup label="Mammals">
    <option value="dog">Dog</option>
    <option value="cat">Cat</option>
  </optgroup>
  <optgroup label="Birds">
    <option value="parrot">Parrot</option>
  </optgroup>
</select>

If no value is set on an option, the text content is submitted instead.

Question 18

What is the purpose of the <select> and <option> tags in a form?

The <select> tag creates a dropdown menu (or a listbox with multiple); <option> defines each choice inside it. They’re always used together.

<select name="size">
  <option value="s">Small</option>
  <option value="m" selected>Medium</option>
  <option value="l">Large</option>
</select>

Key roles:

  • <select> — the control itself. Attributes: name, multiple, size, required, disabled.
  • <option> — one selectable choice. The value is submitted; the text content is displayed.

When to use: short, predefined lists (country, size, category). Avoid for:

  • Very long lists (100+ items) — use a searchable combobox (<input> with <datalist>, or a JS autocomplete) instead.
  • Two-option choices — use radios or a toggle; dropdowns force an extra click.

On mobile, <select> triggers the OS-native picker — usually a better experience than custom dropdown widgets.

Question 19

How do you create a password input field in a form?

A password input is created with <input type="password">. It behaves like a text input but masks characters as they’re typed (usually showing dots or asterisks).

<label for="pwd">Password</label>
<input type="password" id="pwd" name="password" required minlength="8"
       autocomplete="current-password">

Important attributes:

  • autocomplete — tell the browser/password manager what role the field plays:
    • current-password — login page.
    • new-password — sign-up or change-password page.
    • off — discouraged except for security-sensitive flows.
  • minlength — basic strength check (real validation belongs server-side).

Important safeguards:

  • Masking is visual only — it does not encrypt the value. Always use HTTPS so the password is encrypted in transit.
  • Never log password values or store them in plain text — hash with bcrypt, scrypt, or Argon2 on the server.
  • Consider a “show password” toggle (swap the type attribute to text) — it reduces typos and is an accessibility win.
Question 20

What is the difference between the <div> and <span> tags?

Both <div> and <span> are generic containers with no inherent meaning. They’re used for grouping content to apply styling or scripting. The difference is how they flow on the page:

<div><span>
DisplayBlock-levelInline
Line break before/afterYesNo
Common useSections, cards, layout containersBits of text inside a paragraph
<div class="card">
  <p>
    Price: <span class="highlight">$42</span> today only!
  </p>
</div>

Rule of thumb: reach for semantic tags first — <article>, <section>, <nav>, <header>, <main>, etc. Only use <div> when no semantic element fits. <span> is the default choice for styling or scripting a small piece of inline text that doesn’t have a more specific tag (like <em>, <strong>, <code>).

Question 21

How do you add a comment in HTML?

HTML comments are wrapped in <!-- and -->. Anything inside is ignored by the browser and not rendered on the page — though it’s visible to anyone who views the page source.

<!-- This is a single-line comment -->

<!--
  Multi-line comments
  work too. Good for
  documentation and TODOs.
-->

<nav>
  <!-- TODO: add mobile menu -->
  <a href="/">Home</a>
</nav>

Uses:

  • Notes to yourself or teammates about why code is structured a certain way.
  • Temporarily disabling code without deleting it.
  • Marking sections in long files.

Important: comments are sent to the client — never put secrets, API keys, internal URLs, or sensitive business logic in HTML comments. You can’t nest comments (-- inside breaks them), and browsers are strict about the syntax. Build tools can strip comments at deploy time to reduce file size.

Question 22

What is the purpose of the <meta> tag in HTML?

The <meta> tag provides metadata about the document — information about the page itself, rather than content displayed on it. It’s a self-closing tag placed inside <head>.

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <meta name="description" content="Cheap flights to Paris.">
  <meta name="theme-color" content="#1a73e8">

  <!-- Open Graph (social sharing) -->
  <meta property="og:title" content="Paris Deals">
  <meta property="og:image" content="/hero.jpg">
</head>

Common meta tags you should know:

  • charset="UTF-8" — declares character encoding. Put it first in <head>.
  • viewport — essential for responsive/mobile pages. Without it, phones render at desktop width and zoom out.
  • description — the short snippet search engines show under your page title.
  • Open Graph (og:*) and Twitter Card tags — control how links preview on social media.

Meta tags don’t render on the page, but they heavily influence SEO, social previews, and mobile behavior.

Question 23

How do you link an external CSS file to an HTML document?

Link an external CSS file with the <link> tag, placed inside <head>. Three attributes matter: rel (the relationship), href (the file path), and optionally media (when the styles apply).

<head>
  <link rel="stylesheet" href="styles.css">

  <!-- Separate print-only stylesheet -->
  <link rel="stylesheet" href="print.css" media="print">

  <!-- Remote stylesheet from a CDN -->
  <link rel="stylesheet" href="https://cdn.example.com/reset.css">
</head>

Key notes:

  • rel="stylesheet" is required — it tells the browser this is a CSS file.
  • Placement: inside <head> so styles load before the page renders (avoids the “flash of unstyled content”).
  • Path: can be relative (styles.css, /css/main.css) or absolute (https://...).
  • Multiple files: add multiple <link> tags — they load in order; later rules can override earlier ones.

For small, page-specific styles you can use the <style> tag inline, but for anything reusable, external CSS is cleaner and cacheable.

Question 24

How do you link an external JavaScript file to an HTML document?

Link external JavaScript with the <script> tag and a src attribute. Placement and loading behavior matter for performance.

<!-- Classic placement: end of <body> so HTML parses first -->
<script src="app.js"></script>

<!-- Modern: in <head> with defer or async -->
<script src="app.js" defer></script>
<script src="analytics.js" async></script>

<!-- ES modules -->
<script type="module" src="main.js"></script>

Loading strategies:

AttributeWhen it runsWhen to use
NoneBlocks parsing; runs immediatelyRarely needed; avoid.
deferDownloads in parallel; runs after HTML parses, in orderApp code that needs the DOM ready.
asyncDownloads in parallel; runs ASAP, no order guaranteeIndependent scripts like analytics.
type="module"Deferred by default; supports import/exportModern ES modules.

A <script> tag with src must still have a closing </script>. The tag is never self-closing — common source of silent bugs.

Question 25

What is the purpose of the <head> tag in an HTML document?

The <head> tag contains metadata and resources — information about the page rather than content shown to the user. It sits before <body>.

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>My Page</title>
    <meta name="description" content="A brief page description">
    <link rel="icon" href="/favicon.ico">
    <link rel="stylesheet" href="styles.css">
    <script src="app.js" defer></script>
  </head>
  <body>...</body>
</html>

Typical contents:

  • <title> — shown in the browser tab and search results.
  • <meta> tags — charset, viewport, description, social sharing.
  • <link> — stylesheets, favicons, preloads.
  • <script> — JavaScript (preferably with defer).
  • <style> — inline CSS for the page.

Almost nothing in <head> is rendered visually — its job is to configure the page before the browser starts drawing.

Question 26

What is the purpose of the <body> tag in an HTML document?

The <body> tag contains everything the user actually sees on the page — text, images, links, forms, videos, buttons, and so on. It comes after <head> and is the second (and last) direct child of <html>.

<!DOCTYPE html>
<html lang="en">
  <head>
    <title>My Page</title>
  </head>
  <body>
    <h1>Welcome!</h1>
    <p>This text shows up on the page.</p>
    <img src="hero.jpg" alt="Hero image">
  </body>
</html>

Think of it this way:

  • <head> = backstage (configuration, resources, metadata).
  • <body> = the stage (visible content).

Global event handlers like onload can be set on <body>, but modern code attaches handlers via JavaScript instead. Every HTML document has exactly one <body>, and you should never nest another inside it.

Question 27

How do you create a heading in HTML?

Headings are created with the <h1> through <h6> tags, where <h1> is the most important and <h6> the least. They visually render larger/bolder by default, but their primary purpose is document structure.

<h1>Site or page title</h1>
<h2>Major section</h2>
<h3>Subsection</h3>
<h4>Nested subsection</h4>

Best practices:

  • One <h1> per page — it should describe the overall topic.
  • Don’t skip levels — go h2 → h3, not h2 → h4. Screen readers and search engines use the hierarchy to understand structure.
  • Never choose headings for size alone. Need big text? Use CSS. Choose the heading level that reflects meaning.

Good heading structure is one of the easiest wins for accessibility and SEO — it lets users with screen readers jump section-by-section, and it helps Google understand what your page is about.

Question 28

How many heading levels are there in HTML?

HTML has six heading levels: <h1>, <h2>, <h3>, <h4>, <h5>, and <h6>. They indicate importance in descending order.

<h1>Chapter title</h1>      <!-- most important -->
<h2>Section</h2>
<h3>Subsection</h3>
<h4>Sub-subsection</h4>
<h5>Rarely used</h5>
<h6>Least important</h6>    <!-- rarely used -->

In practice, most pages only use h1–h3. Levels h5 and h6 are rarely needed — if your document has that many levels, it may be too deeply nested and hard to follow.

💡
The six levels have been part of HTML since 1991 — they predate CSS. That’s why their default sizes decrease visually, but that’s a side effect, not the purpose. Use them for hierarchy, not for styling.
Question 29

How do you create a paragraph in HTML?

A paragraph is created with the <p> tag. It’s a block-level element, so browsers automatically add vertical spacing before and after.

<p>This is the first paragraph. Text wraps
to fit the container automatically.</p>

<p>This is the second. Separated by visible spacing.</p>

Things to know:

  • Line breaks and extra spaces in your source don’t show up. HTML collapses whitespace. Use <br> for a forced line break within a paragraph, or a new <p> for a new thought.
  • Paragraphs can’t contain block-level elements — don’t put a <div> or <ul> inside a <p>. The browser will silently close the paragraph early, which breaks layout.
  • For inline emphasis use <em>, <strong>, <code>, or <span> within a paragraph.

Styling (margins, font size, line height) is handled with CSS — <p> just declares the structural intent that this is a paragraph of text.

Question 30

How do you create a line break in HTML?

A line break is inserted with the self-closing <br> tag. It forces the next content onto a new line without starting a new paragraph.

<p>
  123 Main Street<br>
  Springfield, IL 62701<br>
  United States
</p>

<p>
  Roses are red,<br>
  Violets are blue.
</p>

When to use <br>: where the line break itself is meaningful — inside addresses, poems, song lyrics, signatures. These are cases where the text structure actually is broken across lines.

When NOT to use <br>:

  • To add space between paragraphs — that’s CSS’s job (margin).
  • To separate items — use a list (<ul>, <ol>) instead.
  • To fake layout — use CSS (flexbox, grid, padding).

Overusing <br> is a classic sign of a beginner HTML file. A handful per page is fine; dozens usually indicates the structure needs rethinking.

Question 31

What is the purpose of the <hr> tag in HTML?

The <hr> tag inserts a horizontal rule — a thin dividing line across the page. Semantically, it represents a thematic break: a shift in subject or a scene change within a section.

<p>End of the first topic.</p>
<hr>
<p>Beginning of a new topic.</p>

It’s a self-closing tag with no content. Originally <hr> was purely presentational; HTML5 re-defined it as semantic (the “break” meaning) and delegated its appearance to CSS.

hr {
  border: none;
  height: 1px;
  background: #ccc;
  margin: 2rem 0;
}

When to use: a genuine topic shift — separating the end of a blog post from author info, or dividing chapters in a long article. When NOT to use: purely decorative lines (use CSS borders on surrounding elements) or to separate list items (a list’s structure already implies separation).

Question 32

How do you create a bold text in HTML?

There are two tags for making text bold. They look identical by default but have different meanings:

  • <strong> — indicates strong importance. Screen readers may change their tone. Use this when the bold text is genuinely significant.
  • <b> — marks text for visual stylistic offset without implying importance (product names, keywords in a summary). Purely presentational.
<p><strong>Warning:</strong> this action cannot be undone.</p>

<p>The ship <b>HMS Beagle</b> sailed in 1831.</p>

Rule of thumb: prefer <strong>. It conveys meaning to assistive tech and search engines; <b> doesn’t.

💡
For purely visual bolding with no semantic weight, the cleanest modern approach is CSS: font-weight: bold on a <span> or a styled class.
Question 33

How do you create an italic text in HTML?

Like bold, italic text has two tags with similar appearance but different meanings:

  • <em> — marks emphasis. Screen readers often change inflection. Use when the meaning changes if you remove the stress.
  • <i> — italicizes text that is set off from normal prose — titles of works, foreign words, technical terms, thoughts — without implying emphasis.
<p>I <em>really</em> don’t like mushrooms.</p>

<p>She was reading <i>Pride and Prejudice</i> again.</p>
<p>The French term for it is <i lang="fr">bonjour</i>.</p>

In doubt, use <em> — it’s the semantic default for italicized meaning. Reserve <i> for conventions like book titles and foreign terms.

For purely decorative italics, CSS (font-style: italic) is the right approach.

Question 34

How do you create an underline text in HTML?

The <u> tag creates underlined text. But in modern web design it’s used cautiously — most users associate underlines with clickable links, so arbitrary underlining can confuse them.

<p>The word <u>teh</u> is a misspelling of “the”.</p>

Semantically, <u> marks text with a non-textual annotation — originally for proper nouns in Chinese text, misspellings, or similar conventions that are rendered with underlines.

Usually better alternatives:

  • For emphasis — use <em> (italic) or <strong> (bold).
  • For spelling corrections — use <u> combined with a CSS class like text-decoration-style: wavy.
  • For purely visual underlines — use CSS: text-decoration: underline.
<style>
  .misspelled { text-decoration: underline wavy red; }
</style>
<p>I went <u class="misspelled">thier</u> yesterday.</p>

Whenever you underline text, ensure it’s visually distinct from your link styles.

Question 35

How do you create a strikethrough text in HTML?

Strikethrough text has two semantic options, used for different situations:

  • <s> — content that is no longer accurate or relevant. Think old prices, outdated facts, discontinued items.
  • <del> — content that has been deleted in a document edit. Often paired with <ins> (inserted text) in change-tracking.
<p>Was <s>$99</s> now $49!</p>

<p>
  The meeting is at <del>3pm</del> <ins>4pm</ins> tomorrow.
</p>

Both render with a line through them by default. The difference is in meaning: <s> is about accuracy, <del> is about editorial history.

For a purely visual line-through with no semantic meaning, CSS is the cleaner choice:

.strikethrough { text-decoration: line-through; }

The older <strike> tag is deprecated and should not be used in new code.

Question 36

What is the purpose of the <em> tag in HTML?

The <em> tag marks text as emphasized — text whose meaning is stressed or whose removal changes the meaning of the sentence. Browsers render it italic by default; screen readers may alter their pitch or emphasis.

<p>I <em>told</em> you it would rain.</p>
<p>I told <em>you</em> it would rain.</p>
<p>I told you it would <em>rain</em>.</p>

Notice how each sentence has a different meaning depending on what is emphasized — that’s exactly what <em> captures.

<em> vs <i>:

  • <em> = emphasis (stressed speech). Meaning-bearing.
  • <i> = italic text for conventional reasons (titles, foreign words) without implying emphasis.

You can nest <em> tags to indicate greater emphasis, though this is rare in practice:

<p>This is <em>really <em>really</em> important</em>.</p>
Question 37

What is the purpose of the <strong> tag in HTML?

The <strong> tag marks text as having strong importance, seriousness, or urgency. Browsers render it bold by default, but the meaning is what matters — not the visual style.

<p><strong>Warning:</strong> this file will be deleted permanently.</p>

<p>Read the <strong>terms and conditions</strong> before signing.</p>

<strong> vs <b>:

  • <strong> = importance/severity. Conveys meaning to screen readers and search engines.
  • <b> = visual boldness without implied importance (product names, keywords).

Use <strong> for:

  • Safety warnings, error messages, critical instructions.
  • Important labels in forms (“Required”).
  • Keywords users must remember in step-by-step guides.

Nested <strong> increases importance level (valid but rarely used in practice). For pure visual boldness that carries no special meaning, prefer CSS font-weight: bold.

Question 38

How do you create a blockquote in HTML?

The <blockquote> tag marks up a quoted block of text, usually multiple lines. Browsers indent it by default. For inline (single-sentence) quotations, use <q> instead.

<blockquote cite="https://example.com/speech">
  <p>The only thing we have to fear is fear itself.</p>
  <footer>— Franklin D. Roosevelt</footer>
</blockquote>

<p>
  She said <q>come back later</q> and closed the door.
</p>

Useful attributes and patterns:

  • cite — URL of the source. It’s machine-readable but not visually shown; pair it with a visible link or <cite> tag for the source name.
  • <cite> — wrap the title of the cited work (book, article, speech).
  • <footer> inside the blockquote is a common place for attribution.

<blockquote> is for quotations, not for generic indentation or stylistic emphasis. If you just want an indented container, use CSS (margin-left or padding-left).

Question 39

How do you create a hyperlink that opens in a new tab?

To open a link in a new tab, add target="_blank" to the <a> tag. Always pair it with rel="noopener" for security.

<a href="https://example.com" target="_blank" rel="noopener">
  Open Example in a new tab
</a>

Values for target:

  • _blank — new tab/window.
  • _self — same tab (the default).
  • _parent / _top — relevant only inside iframes.

Why rel="noopener" matters: without it, the newly-opened page has a reference back to yours (window.opener) and can manipulate your original page. noopener severs that link. Add noreferrer as well if you want to hide which site the user came from:

<a href="https://external.com" target="_blank" rel="noopener noreferrer">
  External link
</a>

Modern browsers apply noopener automatically for target="_blank", but writing it explicitly works in older browsers too and signals your intent.

Question 40

What is the purpose of the <title> tag in HTML?

The <title> tag sets the document title, placed inside <head>. Its content appears in several important places, even though it never shows on the page itself.

<head>
  <title>Contact Us – Acme Corp</title>
</head>

Where the title appears:

  • Browser tab / window bar — the visible label of your page.
  • Bookmarks — the default name when a user saves the page.
  • Search engine results — Google, Bing, etc. use it as the clickable headline.
  • Social media cards — fallback when no Open Graph title is set.

Guidelines:

  • Keep it under ~60 characters — longer titles get truncated in search results.
  • Put the most important info first (page topic before site name).
  • Make every page’s title unique — “Home” on every page is bad for usability and SEO.

Every HTML document must have exactly one <title>. It’s required by the HTML specification.

Question 41

How do you create a numbered list in HTML?

A numbered list is the same as an ordered list — use the <ol> tag with <li> items. The browser numbers them automatically, so you don’t type the numbers yourself.

<ol>
  <li>First step</li>
  <li>Second step</li>
  <li>Third step</li>
</ol>

Renders as:

  1. First step
  2. Second step
  3. Third step

If you reorder, add, or remove items, numbering updates automatically — no manual fixing. You can customize the numbering with attributes:

<ol start="5" type="i" reversed>
  <li>Labeled v (counting down)</li>
  <li>iv</li>
  <li>iii</li>
</ol>

If you want bullet points instead of numbers, use <ul> (unordered list). If the order is meaningful — like instructions, rankings, or event timelines — <ol> is the right choice because the sequence carries meaning.

Question 42

What is the difference between the <ol> and <ul> tags?

Both tags create lists with <li> items inside. The difference is about whether the order matters:

<ol> (ordered)<ul> (unordered)
MeaningSequence mattersSequence doesn’t matter
Default markerNumbers: 1, 2, 3Bullets: •
Use forSteps, rankings, recipes, instructionsFeatures, tags, navigation, groupings
<!-- Ordered: rearranging changes meaning -->
<ol>
  <li>Whisk eggs</li>
  <li>Add milk</li>
  <li>Cook on low heat</li>
</ol>

<!-- Unordered: reordering is fine -->
<ul>
  <li>Eggs</li>
  <li>Milk</li>
  <li>Butter</li>
</ul>

The quick test: if you shuffled the items, would the list still make sense? If yes, use <ul>. If the order matters, use <ol>. Screen readers announce “numbered list of 5 items” vs “list of 5 items,” which helps users understand structure.

Question 43

How do you create a list item in HTML?

Every item inside a list — whether ordered or unordered — is wrapped in a <li> (list item) tag. It’s always a child of <ol>, <ul>, or <menu>.

<ul>
  <li>Apple</li>
  <li>Banana</li>
  <li>Cherry</li>
</ul>

List items can hold almost any content — paragraphs, links, images, even other lists:

<ul>
  <li>
    <strong>Groceries</strong>
    <ul>
      <li>Milk</li>
      <li>Eggs</li>
    </ul>
  </li>
  <li><a href="/todo">Check todo list</a></li>
</ul>

Inside an <ol>, you can override numbering per item with the value attribute:

<ol>
  <li>First</li>
  <li value="10">Jumps to 10</li>
  <li>Continues at 11</li>
</ol>

<li> outside a list container isn’t valid HTML — always wrap it.

Question 44

How do you create a nested list in HTML?

A nested list is just a list placed inside another list’s <li>. It must be inside the <li>, not directly inside the outer <ul> or <ol>.

<ul>
  <li>Fruits
    <ul>
      <li>Apple</li>
      <li>Banana</li>
    </ul>
  </li>
  <li>Vegetables
    <ul>
      <li>Carrot</li>
      <li>Spinach</li>
    </ul>
  </li>
</ul>

You can mix list types:

<ol>
  <li>Prepare ingredients
    <ul>
      <li>Flour</li>
      <li>Sugar</li>
    </ul>
  </li>
  <li>Mix and bake</li>
</ol>

Common mistakes to avoid:

  • Placing a nested list directly inside <ul>/<ol> (not inside an <li>) is invalid HTML. The browser will still render something, but it may look wrong in screen readers.
  • Deeply nested lists become hard to read — try to keep nesting at 2–3 levels max.
Question 45

How do you add a background image to an HTML element?

Background images aren’t added with HTML — they’re applied with CSS. The background-image property works on any element.

<div class="hero">Welcome!</div>

<style>
  .hero {
    background-image: url('hero.jpg');
    background-size: cover;            /* fill the element */
    background-position: center;       /* center the image */
    background-repeat: no-repeat;      /* don’t tile it */
    height: 400px;
  }
</style>

You can also stack a gradient overlay on top of an image:

.hero {
  background-image:
    linear-gradient(rgba(0,0,0,.4), rgba(0,0,0,.4)),
    url('hero.jpg');
  background-size: cover;
}

When not to use a background image:

  • If the image conveys content (like a product photo or a chart), use <img> — background images are invisible to screen readers and skipped by search engines.
  • Background images are purely decorative. Save them for aesthetics (hero banners, textures, gradients).

The old HTML background attribute on <body> is long deprecated — CSS is the modern way.

Question 46

How do you add a tooltip to an HTML element?

A tooltip (small popup shown on hover) is added with the title attribute. It’s a global attribute, so it works on almost any HTML element.

<button title="Save your changes">Save</button>

<a href="/help" title="Opens the help center">Help</a>

<img src="chart.png" alt="Sales chart" title="Q4 2025 revenue breakdown">

The browser displays the title text as a small tooltip after the user hovers for ~1 second.

Caveats:

  • Not accessible to touch-screen users — no hover on phones/tablets. Don’t hide essential info in a title.
  • Keyboard users may not see the tooltip — behavior varies by browser.
  • Screen readers often ignore title or announce it inconsistently.
  • You can’t style the default tooltip appearance.

For important hints or richer tooltips (custom styling, always-visible on focus), build one with CSS and a bit of JavaScript, or use an accessible library. The native title attribute is fine for minor hints but a poor primary UX pattern.

Question 47

How do you create a navigation menu in HTML?

A navigation menu is typically built with the semantic <nav> element wrapping a list (<ul>) of links. The list makes the menu structured and screen-reader-friendly.

<nav aria-label="Main">
  <ul>
    <li><a href="/">Home</a></li>
    <li><a href="/products">Products</a></li>
    <li><a href="/pricing">Pricing</a></li>
    <li><a href="/contact">Contact</a></li>
  </ul>
</nav>

CSS removes the default bullets and arranges items horizontally:

nav ul {
  list-style: none;
  display: flex;
  gap: 1.5rem;
  padding: 0;
}
nav a { text-decoration: none; color: #333; }

Best practices:

  • Use one <nav> for each major nav area (main site nav, footer links, in-page table of contents) and label each with aria-label.

  • Mark the current page visually and with aria-current="page" on its link.
Question 48

What is the purpose of the <nav> tag in HTML?

The <nav> tag marks a section of the page as a block of navigation links — a primary menu, a table of contents, or major link groups in a footer. It’s one of several HTML5 semantic landmarks.

<nav aria-label="Main">
  <a href="/">Home</a>
  <a href="/about">About</a>
  <a href="/contact">Contact</a>
</nav>

Why it matters:

  • Screen readers let users jump directly to <nav> landmarks, skipping repetitive site chrome.
  • Search engines understand which links represent site structure vs content links.
  • Readability — clear semantic meaning for anyone reading the markup.

Guidelines:

  • A page can have multiple <nav> elements (main nav, sidebar, footer, etc.). Label each with aria-label or aria-labelledby to distinguish them.
  • Not every group of links needs <nav> — reserve it for the major navigation. Inline links in an article should stay plain.
  • The footer’s site-wide links may be wrapped in a <nav>; legal notices and copy generally don’t need it.
Question 49

How do you create a footer in HTML?

A footer is created with the HTML5 semantic <footer> tag. It typically sits at the bottom of the page and holds metadata about its parent section — copyright, contact info, social links, sitemap.

<footer>
  <p>© 2026 Acme Corp. All rights reserved.</p>

  <nav aria-label="Footer">
    <a href="/privacy">Privacy</a>
    <a href="/terms">Terms</a>
    <a href="/contact">Contact</a>
  </nav>
</footer>

A page can have more than one footer — each <article> or <section> can have its own footer (author info, article timestamp, etc.):

<article>
  <h2>Blog post title</h2>
  <p>...</p>
  <footer>
    <p>By <a href="/author">Jane Doe</a> on <time datetime="2026-04-20">April 20, 2026</time></p>
  </footer>
</article>

Visual placement (sticky-to-bottom, styled background) is entirely up to CSS. The tag is about meaning, not position.

Question 50

What is the purpose of the <footer> tag in HTML?

The <footer> tag marks a footer section for its nearest ancestor — the whole page, or an article, section, or aside. It usually contains metadata: author, copyright, related links, contact info.

<footer>
  <p>© 2026 Acme. <a href="/privacy">Privacy Policy</a></p>
</footer>

Why it’s useful:

  • Accessibility: screen readers treat <footer> as a contentinfo landmark when it’s the page’s main footer — users can jump to it directly.
  • SEO: search engines understand structure better, distinguishing navigation from incidental info.
  • Clarity: the markup describes intent, not just layout.

What goes in a typical page footer:

  • Copyright notice.
  • Secondary navigation (privacy, terms, careers).
  • Social media icons.
  • Newsletter signup.
  • Contact info.

Sectional footers (inside articles) typically hold author credits, publication dates, or related tags. Footers don’t have to be at the visual bottom — CSS handles layout — but semantically they conclude their parent.

Question 51

How do you create a header in HTML?

A header is created with the HTML5 semantic <header> tag. It typically wraps introductory content — a site logo, page title, main navigation, or article metadata.

<header>

  <nav aria-label="Main">
    <a href="/">Home</a>
    <a href="/products">Products</a>
    <a href="/contact">Contact</a>
  </nav>

<article>

  </header>

</article>

The name <header> describes meaning, not position — visually, a header normally sits at the top, but the tag’s purpose is to group introductory content. CSS handles the visual placement.

Question 52

What is the purpose of the <header> tag in HTML?

The <header> tag represents introductory content for its nearest ancestor — the page, an article, a section, or an aside. Typical contents: site title, navigation, search form, logo, author byline.

<header>
  <h1>My Blog</h1>
  <nav><a href="/">Home</a></nav>
</header>

Why it matters:

  • Accessibility: when <header> is a direct child of <body>, screen readers expose it as a banner landmark — users can jump straight to it.
  • Clarity of structure: it tells readers of the code (and AI tools, and search engines) which part of the page is introductory.
  • Reusable within the document: each article or section can have its own internal <header>.

Guidelines:

  • Don’t nest a <header> inside another <header> or inside a <footer>.
  • If the top bar is purely a logo image, a plain <div> or <img> might be enough — reserve <header> for content with a heading or introductory role.
Question 53

How do you create a section in HTML?

A section of related content is created with the <section> tag. It groups content that belongs together thematically — typically under a shared heading.

<section>
  <h2>Features</h2>
  <p>Our product offers three main features...</p>
</section>

<section>
  <h2>Pricing</h2>

How do you create a hyperlink to an email address in HTML?

  sizes="(max-width: 600px) 100vw, 50vw"
  alt="Mountain landscape">

How it works:

  • srcset lists image files with their widths (400w = 400 pixels wide).
  • sizes tells the browser how wide the image will display at different viewport sizes.
  • The browser picks the smallest file that looks good at the current display width and device pixel ratio.
  • src remains as a fallback for browsers that don’t support srcset.

For simpler responsive sizing without art direction, CSS is enough:

img { max-width: 100%; height: auto; }

This makes any image shrink to fit its container while keeping proportions. Combine CSS sizing with srcset for the best balance of quality, bandwidth, and performance — mobile users on cellular data thank you.

Question 83

What is the purpose of the <picture> tag in HTML?

The <picture> tag gives you fine-grained control over which image is shown — based on screen size, format support, or device pixel density. It contains one or more <source> elements and a final <img> as fallback.

<picture>
  <source media="(max-width: 600px)" srcset="small.jpg">
  <source media="(max-width: 1200px)" srcset="medium.jpg">
  <img src="large.jpg" alt="Product">
</picture>

Main uses:

  • Art direction — show a differently-cropped image on mobile vs desktop (portrait vs landscape).
  • Format selection — serve modern formats (AVIF, WebP) to browsers that support them, JPEG to older ones.
  • Resolution selection — like srcset on <img>, but with more control.
<picture>
  <source srcset="photo.avif" type="image/avif">
  <source srcset="photo.webp" type="image/webp">
  <img src="photo.jpg" alt="Landscape">
</picture>

The browser picks the first <source> that matches — it supports the format and its media condition is true. If none match, it falls back to the <img>. This is how you serve modern next-gen image formats safely today.

Question 84

How do you use the <source> tag in HTML?

The <source> tag provides alternate media files inside <picture>, <audio>, or <video> elements. The browser picks the first <source> it can play or display.

In <picture>:

<picture>
  <source srcset="hero.avif" type="image/avif">
  <source srcset="hero.webp" type="image/webp">
  <img src="hero.jpg" alt="Banner">
</picture>

In <video>:

<video controls>
  <source src="movie.webm" type="video/webm">
  <source src="movie.mp4" type="video/mp4">
  Your browser doesn’t support video.
</video>

In <audio>:

<audio controls>
  <source src="song.opus" type="audio/ogg">
  <source src="song.mp3" type="audio/mpeg">
</audio>

Common attributes:

  • src — file URL (for audio/video).
  • srcset — one or more image URLs with widths (for picture).
  • type — MIME type so the browser can skip unsupported formats.
  • media — CSS media query (picture only) for art direction.

The order matters — list the most preferred format first. The browser stops at the first compatible source, so put modern formats before fallbacks.

Question 85

How do you create a table header in HTML?

A table header is a cell that labels a row or column. Use the <th> tag instead of <td> for header cells — browsers style them bold and centered by default, and screen readers announce them as headers.

<table>
  <tr>
    <th>Name</th>
    <th>Age</th>
    <th>City</th>
  </tr>
  <tr>
    <td>Alice</td>
    <td>30</td>
    <td>Seattle</td>
  </tr>
</table>

The scope attribute makes headers explicit for accessibility:

<tr>
  <th scope="col">Product</th>
  <th scope="col">Price</th>
</tr>
<tr>
  <th scope="row">Widget</th>
  <td>$9.99</td>
</tr>
  • scope="col" — header applies to the column below.
  • scope="row" — header applies to the row to its right.

Wrap header rows in <thead> for semantic clarity and easier styling. Screen readers use the header-data association to announce column names when reading cells, so users know what each value means.

Question 86

How do you create a table row in HTML?

A table row is created with the <tr> tag. Every cell in a table must be inside a <tr>, and rows stack vertically to form the grid.

<table>
  <tr>
    <th>Item</th>
    <th>Quantity</th>
    <th>Price</th>
  </tr>
  <tr>
    <td>Apple</td>
    <td>3</td>
    <td>$1.50</td>
  </tr>
  <tr>
    <td>Bread</td>
    <td>1</td>
    <td>$3.00</td>
  </tr>
</table>

Rules:

  • <tr> can only contain <th> or <td> cells.
  • Every row should have the same number of cells (use colspan when merging).
  • Rows can go inside <thead>, <tbody>, or <tfoot> for sectioning.

For styling, the :nth-child() CSS selector is perfect for rows:

tr:nth-child(even) { background-color: #f9f9f9; }
tr:hover           { background-color: #e8f4fd; }

Striped rows and hover effects greatly improve readability for wide tables. Keep row structure consistent — varying cell counts per row breaks layout and confuses assistive technology.

Question 87

How do you create a table cell in HTML?

A table cell holds the data inside a row. Use <td> for regular data cells and <th> for header cells.

<table>
  <tr>
    <th>Month</th>
    <th>Sales</th>
  </tr>
  <tr>
    <td>January</td>
    <td>$12,400</td>
  </tr>
  <tr>
    <td>February</td>
    <td>$15,200</td>
  </tr>
</table>

Cell tags:

TagPurposeDefault styling
<td>Regular data cellLeft-aligned, normal weight
<th>Header cellCentered, bold

Useful cell attributes:

  • colspan="2" — cell spans 2 columns.
  • rowspan="3" — cell spans 3 rows.
  • headers="idOfTh" — explicit association to a header (for complex tables).

Cells can contain anything — plain text, images, links, inputs, even nested tables (rarely a good idea). Keep cells focused on single pieces of data; use CSS classes on cells for alignment, padding, or emphasis rather than inline styles.

Question 88

How do you create a table footer in HTML?

A table footer is a section at the bottom of a table for totals, summaries, or notes. Wrap its rows in the <tfoot> element.

<table>
  <thead>
    <tr>
      <th>Product</th>
      <th>Price</th>
    </tr>
  </thead>
  <tbody>
    <tr><td>Book</td><td>$15</td></tr>
    <tr><td>Pen</td><td>$2</td></tr>
  </tbody>
  <tfoot>
    <tr>
      <th>Total</th>
      <td>$17</td>
    </tr>
  </tfoot>
</table>

Benefits of <tfoot>:

  • Semantic meaning — screen readers announce it as a footer, helping users understand that it contains summary data.
  • Print repetition — some browsers repeat <tfoot> on every printed page of a long table.
  • Easier styling — target with tfoot td { font-weight: bold; }.

The <tfoot> can appear anywhere inside the <table> — before or after <tbody> — and the browser still renders it at the bottom. Use it for totals, grand averages, counts, or footnotes that relate to the whole table.

Question 89

What is the purpose of the <thead>, <tbody>, and <tfoot> tags in HTML?

These three tags divide a table into logical sections — header, body, and footer — giving structure to otherwise flat rows.

TagPurpose
<thead>Wraps header rows (column titles)
<tbody>Wraps the main data rows
<tfoot>Wraps footer rows (totals, summaries)
<table>
  <thead>
    <tr><th>Item</th><th>Cost</th></tr>
  </thead>
  <tbody>
    <tr><td>Coffee</td><td>$4</td></tr>
    <tr><td>Muffin</td><td>$3</td></tr>
  </tbody>
  <tfoot>
    <tr><td>Total</td><td>$7</td></tr>
  </tfoot>
</table>

Why use them:

  • Semantic clarity — screen readers announce sections, helping users navigate tables.
  • Scroll + fixed headers — when <tbody> scrolls, <thead> can stick.
  • Print repetition — <thead> and <tfoot> repeat on each printed page.
  • Easier CSS targeting — style headers, body rows, and footers differently without extra classes.

Including these sections is a small extra effort that pays off in accessibility, print layout, and maintainable styling.

Question 90

How do you merge table cells in HTML?

Merge cells using colspan (horizontal merge) or rowspan (vertical merge) on a <td> or <th> — the value is the number of cells it should span.

<table border="1">
  <tr>
    <th colspan="2">Name</th>
    <th>Age</th>
  </tr>
  <tr>
    <td>Jane</td>
    <td>Smith</td>
    <td>28</td>
  </tr>
</table>

Here the “Name” header spans two columns above First and Last name cells.

Row span example:

<table border="1">
  <tr>
    <td rowspan="2">Q1</td>
    <td>January</td>
  </tr>
  <tr>
    <td>February</td>
  </tr>
</table>

Important: when a cell spans multiple rows or columns, skip the cells it covers in subsequent rows. Forgetting to remove them creates extra cells and breaks the layout.

Accessibility tip: spanned headers can confuse screen readers. For complex tables, add scope="colgroup" or scope="rowgroup" to clarify which cells a merged header governs. Keep merges simple — deeply nested merges are hard to maintain and often hint that a single table is doing two separate jobs.

Question 91

How do you create a caption for a table in HTML?

A table caption is a title or description for the whole table. Add it with the <caption> tag, which must be the first child of <table>.

<table>
  <caption>Monthly Sales Report — Q3 2026</caption>
  <thead>
    <tr>
      <th>Month</th>
      <th>Revenue</th>
    </tr>
  </thead>
  <tbody>
    <tr><td>July</td><td>$28,000</td></tr>
    <tr><td>August</td><td>$32,500</td></tr>
  </tbody>
</table>

Benefits:

  • Screen readers announce the caption before reading the table, giving context upfront.
  • Semantic meaning — better than a <h3> above the table because it’s tied to the table itself.
  • Printed documents keep the caption with the table.

By default, the caption appears above the table. You can move it with CSS:

caption {
  caption-side: bottom;
  font-style: italic;
  padding: 0.5em;
}

Keep captions brief but informative — “Sales by Region, 2026” beats “Table 1” because it tells users what they’re looking at without having to scan the content.

Question 92

How do you add a border to a table in HTML?

The HTML border attribute is deprecated — modern tables use CSS for borders. CSS gives far more control over thickness, color, and style.

<table class="bordered">
  <tr><th>Name</th><th>Role</th></tr>
  <tr><td>Ada</td><td>Engineer</td></tr>
  <tr><td>Grace</td><td>Admiral</td></tr>
</table>

<style>
  .bordered {
    border-collapse: collapse;
    width: 100%;
  }
  .bordered th, .bordered td {
    border: 1px solid #ccc;
    padding: 8px 12px;
  }
</style>

Key properties:

  • border: 1px solid #ccc — sets the line around each cell.
  • border-collapse: collapse — merges adjacent cell borders into single lines (the default has two borders between cells, looking doubled).
  • border-spacing: 4px — only applies when border-collapse: separate; controls gap between cells.

For different styles:

table { border: 2px solid #333; }            /* outer border */
td, th { border-bottom: 1px solid #eee; }    /* horizontal lines only */

Keeping borders in CSS means your design can evolve without touching HTML — swap light and dark themes, change thickness globally, or remove borders entirely by editing one rule.

Question 93

How do you create a table with alternating row colors in HTML?

Alternating row colors (“zebra striping”) make wide tables much easier to read — your eye follows a single row across without jumping lines. Use the CSS :nth-child() selector.

<table class="striped">
  <tr><th>Name</th><th>Email</th><th>Role</th></tr>
  <tr><td>Alice</td><td>alice@x.com</td><td>Admin</td></tr>
  <tr><td>Bob</td><td>bob@x.com</td><td>User</td></tr>
  <tr><td>Carol</td><td>carol@x.com</td><td>Admin</td></tr>
  <tr><td>Dave</td><td>dave@x.com</td><td>User</td></tr>
</table>

<style>
  .striped tr:nth-child(even) {
    background-color: #f5f5f5;
  }
  .striped tr:hover {
    background-color: #e8f4fd;
  }
</style>

Useful selectors:

  • tr:nth-child(even) — every even-numbered row.
  • tr:nth-child(odd) — every odd-numbered row.
  • tbody tr:nth-child(even) — stripes only data rows, ignoring the header.

Combine with a hover highlight to make the current row pop out. Pick background colors with enough contrast to be visible but not distracting — a light gray (#f5f5f5 or #fafafa) on white works well for most designs. Avoid loud colors; they fight with the data rather than helping you read it.

Question 94

How do you create an image map in HTML?

An image map makes different parts of a single image clickable, each leading to a different link or action. Create it with an <img> linked to a <map> that contains clickable <area> elements.

<img src="worldmap.png" alt="World map" usemap="#regions">

<map name="regions">
  <area shape="rect" coords="0,0,200,150"
        href="/europe" alt="Europe">
  <area shape="circle" coords="300,200,80"
        href="/africa" alt="Africa">
  <area shape="poly" coords="400,100,500,100,450,200"
        href="/asia" alt="Asia">
</map>

How the pieces link:

  • The image’s usemap="#regions" attribute points to the map’s name="regions".
  • Each <area> defines a hotspot with a shape and coords.
  • Every <area> needs an alt attribute for accessibility.

Shape types:

ShapeCoords format
rectx1, y1, x2, y2 — top-left and bottom-right
circlecx, cy, radius
polyx1, y1, x2, y2, ... — series of points

Image maps are rare today — SVG with clickable paths is more flexible, accessible, and scalable. Use maps only for bitmap images where SVG isn’t practical.

Question 95

What is the purpose of the <map> and <area> tags in HTML?

The <map> and <area> tags together turn regions of an image into clickable hotspots — useful for diagrams, floor plans, or anatomical illustrations where clicking different parts leads to different pages.

TagPurpose
<map>Container that holds one or more clickable areas; referenced by an image’s usemap attribute
<area>Defines a single clickable region with a shape, coordinates, and a target URL
<img src="car.jpg" alt="Car" usemap="#car-parts">

<map name="car-parts">
  <area shape="rect" coords="50,20,150,80"
        href="/parts/hood" alt="Hood">
  <area shape="circle" coords="100,200,30"
        href="/parts/wheel" alt="Wheel">
</map>

Key points:

  • The name on <map> must match the usemap on the image (with # prefix in usemap).
  • Every <area> needs an alt attribute so screen readers announce what each hotspot leads to.
  • Hotspots can overlap — the first matching area wins.

Modern alternatives like SVG with <a> wrapped around shapes, or CSS-positioned transparent buttons over an image, offer more flexibility for responsive design. Use image maps when you need quick clickable zones on a static raster image.

Question 96

How do you create a collapsible section in HTML?

A collapsible section is built natively with <details> and <summary> — no JavaScript needed. The summary is always visible; the rest shows or hides when clicked.

<details>
  <summary>Shipping information</summary>
  <p>Orders ship within 2 business days.
  Free shipping on orders over $50.</p>
</details>

<details open>
  <summary>Return policy</summary>
  <p>Free returns within 30 days of purchase.</p>
</details>

How it works:

  • <summary> is the clickable label — always visible, with a disclosure triangle in most browsers.
  • Everything else inside <details> is the expandable content.
  • Add the open attribute to have a section expanded by default.

Why use it:

  • Zero JavaScript — built-in expand/collapse behavior.
  • Keyboard-accessible out of the box (Enter and Space toggle it).
  • Announced correctly by screen readers as an expandable widget.
  • Remembers state when printing (open sections print; closed don’t).

Perfect for FAQs, “show more” sections, advanced options, or any content the user shouldn’t see until they ask for it. Style with CSS; the default arrow can be replaced using the ::marker or summary::-webkit-details-marker pseudo-elements.

Question 97

How do you create an accordion in HTML?

An accordion is a stack of collapsible sections where only one (or sometimes several) is open at a time. You can build a simple accordion natively with multiple <details> elements.

<div class="accordion">
  <details name="faq">
    <summary>How do I reset my password?</summary>
    <p>Click “Forgot password” on the login page.</p>
  </details>

  <details name="faq">
    <summary>How do I cancel my subscription?</summary>
    <p>Go to Account Settings and click Cancel.</p>
  </details>

  <details name="faq">
    <summary>What payment methods are accepted?</summary>
    <p>Credit card, PayPal, and Apple Pay.</p>
  </details>
</div>

The name attribute is the key feature — when multiple <details> share the same name, opening one automatically closes the others. This gives you exclusive, radio-button-style behavior without any JavaScript.

Simple styling:

.accordion details {
  border-bottom: 1px solid #ddd;
  padding: 1em 0;
}
.accordion summary {
  cursor: pointer;
  font-weight: bold;
}

For richer accordions with smooth animations or multi-level nesting, frameworks like Radix UI or Headless UI provide fully accessible components. Native <details> with name is ideal for FAQs and simple toggle groups.

Question 98

How do you create a tabbed navigation in HTML?

A tabbed navigation lets users switch between panels of content by clicking labels. HTML alone doesn’t offer a native tabs widget, so you build one with a <ul> of tab buttons and content sections that show or hide.

<div class="tabs">
  <ul role="tablist">
    <li><button role="tab" aria-selected="true"  aria-controls="p1">Overview</button></li>
    <li><button role="tab" aria-selected="false" aria-controls="p2">Details</button></li>
    <li><button role="tab" aria-selected="false" aria-controls="p3">Reviews</button></li>
  </ul>

  <section id="p1" role="tabpanel">Overview content here.</section>
  <section id="p2" role="tabpanel" hidden>Product details here.</section>
  <section id="p3" role="tabpanel" hidden>Reviews here.</section>
</div>

ARIA roles for accessibility:

  • role="tablist" on the container of buttons.
  • role="tab" on each tab button.
  • role="tabpanel" on each content section.
  • aria-controls links a tab to its panel.
  • aria-selected marks the active tab.

JavaScript toggles the hidden attribute and aria-selected when a tab is clicked. For production, use a component library (Radix, Headless UI, Material UI) — correct tab behavior (arrow-key navigation, focus management) is surprisingly tricky to get right by hand.

Question 99

How do you create a modal dialog in HTML?

The <dialog> element creates a native modal dialog in modern HTML — with proper focus trapping, backdrop, and Esc-to-close built in.

<dialog id="confirmModal">
  <h2>Delete this item?</h2>
  <p>This action cannot be undone.</p>
  <form method="dialog">
    <button value="cancel">Cancel</button>
    <button value="confirm">Delete</button>
  </form>
</dialog>

<button onclick="confirmModal.showModal()">Delete</button>

Key methods:

  • .showModal() — opens as a true modal; background is dimmed and blocked.
  • .show() — opens as a non-modal (user can still interact with the page).
  • .close(returnValue) — closes the dialog, optionally returning a value.

Why <dialog> is great:

  • Automatic focus management — keyboard users can’t Tab out of the dialog while modal.
  • Esc key closes it by default.
  • The ::backdrop pseudo-element lets you style the dim overlay.
  • Screen readers announce it correctly as a dialog.
dialog::backdrop {
  background: rgba(0, 0, 0, 0.5);
}

Before <dialog>, developers built modals with positioned <div> elements and JavaScript — correct focus trapping alone was hard. Use the native element whenever possible.

Question 100

How do you create a breadcrumb navigation in HTML?

Breadcrumb navigation shows where the user is within a site hierarchy — like “Home > Products > Laptops > Dell XPS 15.” Build it with an ordered list inside a <nav>, marked with aria-label for screen readers.

<nav aria-label="Breadcrumb">
  <ol class="breadcrumb">
    <li><a href="/">Home</a></li>
    <li><a href="/products">Products</a></li>
    <li><a href="/products/laptops">Laptops</a></li>
    <li aria-current="page">Dell XPS 15</li>
  </ol>
</nav>

<style>
  .breadcrumb {
    display: flex;
    list-style: none;
    padding: 0;
    gap: 0.5em;
  }
  .breadcrumb li + li::before {
    content: "\203A";    /* › separator */
    margin-right: 0.5em;
    color: #888;
  }
  .breadcrumb [aria-current="page"] {
    color: #666;
    font-weight: bold;
  }
</style>

Best practices:

  • Use an ordered list (<ol>) — order matters.
  • Wrap in <nav> with aria-label="Breadcrumb" so screen readers announce it.
  • Mark the current page with aria-current="page" and omit the link (you’re already there).
  • Use a CSS ::before pseudo-element for separators instead of typing / or > into the markup — screen readers skip decorative characters this way.

Breadcrumbs help users understand site structure and quickly jump up levels — especially valuable for e-commerce and documentation sites with deep hierarchies.
