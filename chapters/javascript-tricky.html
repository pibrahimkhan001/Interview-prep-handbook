Question 2

Can you explain how JavaScript's event loop works and how it handles asynchronous operations?

The event loop is the mechanism that lets single-threaded JavaScript handle asynchronous work without blocking. It continuously checks: is the call stack empty? If yes, pull the next task from a queue and run it.

Key components

  • Call stack: currently executing functions.
  • Microtask queue: Promise callbacks (.then, await continuations), queueMicrotask, MutationObserver. Drained fully after every task.
  • Macrotask queue (task queue): setTimeout, setInterval, I/O, UI events.
console.log("1");
setTimeout(() => console.log("2"), 0);
Promise.resolve().then(() => console.log("3"));
console.log("4");
// Output: 1, 4, 3, 2  — microtasks run before the next macrotask

Each tick: run one macrotask → drain all microtasks → render (maybe) → repeat.

Question 3

What are the potential pitfalls of using == versus ===?

== does type coercion; === checks value and type.

0 == "0"       // true — string coerced
0 == []        // true — [] → "" → 0
"0" == []      // false! — "0" vs ""
null == undefined  // true (special case)
null === undefined // false
NaN == NaN     // false (NaN is never equal to anything)

Pitfalls

  • Coercion rules are not transitive: "0" == 0 and 0 == [] but "0" != [].
  • Hidden bugs when checking form input: if (value == 0) also matches "", false, [].
💡
Default to ===. The one acceptable use of == is x == null to catch both null and undefined.
Question 4

How do closures work in JavaScript, and what are some common uses for them?

A closure is a function bundled with a reference to its lexical scope. The inner function retains access to outer variables even after the outer function has returned.

function makeCounter() {
  let count = 0;            // captured by closure
  return () => ++count;     // inner keeps seeing `count`
}
const c = makeCounter();
c(); c(); c(); // 1, 2, 3

Common uses

  • Data privacy: variables outside the returned function are unreachable externally.
  • Partial application / currying: pre-bake arguments.
  • Callbacks / event handlers: preserve state between invocations.
  • Module pattern: expose only the public API.
⚠️
Loop gotcha: for (var i = 0; i < 3; i++) setTimeout(() => console.log(i)) logs 3,3,3 — one shared i. Use let to get a fresh binding per iteration.
Question 5

Explain the concept of prototypes and how inheritance works in JavaScript.

Every object has an internal [[Prototype]] (accessible via Object.getPrototypeOf(obj) or the legacy __proto__). Property lookups walk up this chain until the property is found or the chain ends at null.

const animal = { eats: true };
const rabbit = Object.create(animal);
rabbit.jumps = true;

rabbit.eats;  // true — inherited from animal
rabbit.jumps; // true — own property

// Constructor-based inheritance
function Animal(name) { this.name = name; }
Animal.prototype.speak = function() { return this.name; };

function Dog(name) { Animal.call(this, name); }
Dog.prototype = Object.create(Animal.prototype);
Dog.prototype.constructor = Dog;

ES6 class is syntactic sugar over this exact prototype mechanism — extends sets up the prototype chain, super() calls the parent constructor.

Question 6

How can you create a deep copy of an object in JavaScript?

A deep copy duplicates nested objects/arrays so mutations to the copy don't affect the original.

// 1. structuredClone — modern, handles Date, Map, Set, cycles
const copy = structuredClone(original);

// 2. JSON round-trip — loses functions, Date → string, no cycles
const copy = JSON.parse(JSON.stringify(original));

// 3. Manual recursion — full control
function deepClone(v, seen = new WeakMap()) {
  if (v === null || typeof v !== "object") return v;
  if (seen.has(v)) return seen.get(v);
  const out = Array.isArray(v) ? [] : {};
  seen.set(v, out);
  for (const k in v) out[k] = deepClone(v[k], seen);
  return out;
}

Pick: structuredClone for modern environments (Node 17+, all evergreen browsers). JSON only for plain data. Lodash _.cloneDeep for maximum compatibility.

Question 8

Can you describe the module pattern in JavaScript and provide an example?

The module pattern uses an IIFE + closure to expose a public API while keeping internals private. Pre-ES6, this was the standard way to get encapsulation.

const Counter = (function() {
  let count = 0;             // private
  function log() { console.log(count); }  // private

  return {
    increment() { count++; log(); },
    reset()     { count = 0; },
    get value() { return count; }
  };
})();

Counter.increment();   // 1
Counter.value;         // 1
Counter.count;         // undefined — truly private

Modern code uses ES modules (export/import) instead — exports are explicit and tooling-friendly. But the pattern still appears in legacy code and in single-file browser scripts.

Question 9

How does the this keyword behave in different contexts?

this is determined by how a function is called, not where it's defined (except arrow functions).

// 1. Method call — `this` is the object before the dot
obj.method();       // this === obj

// 2. Function call — strict: undefined, non-strict: globalThis
fn();               // this === undefined (strict mode)

// 3. Constructor call — `this` is the new object
new Cls();          // this === new instance

// 4. Explicit binding
fn.call(ctx);       // this === ctx
fn.bind(ctx)();     // this === ctx

// 5. Arrow function — inherits `this` from enclosing lexical scope
const arrow = () => this;  // whatever `this` is here

// 6. Event handler — the element (unless arrow)
btn.onclick = function() { this; }   // btn
btn.onclick = () => this;           // outer this
⚠️
Detaching a method loses binding: const m = obj.method; m(); — this is no longer obj.
Question 10

What is the significance of the bind, call, and apply methods in JavaScript?

All three control the value of this (and arguments) when invoking a function.

function greet(greeting, punct) {
  return `${greeting}, ${this.name}${punct}`;
}
const user = { name: "Ibrahim" };

greet.call(user, "Hi", "!");      // immediate, args listed
greet.apply(user, ["Hi", "!"]);   // immediate, args array
const bound = greet.bind(user, "Hi");  // returns new function
bound("!");                       // calls later

Memory mnemonic: Apply takes an Array. Arrow functions ignore all three (lexical this).

Question 11

Explain the concept of hoisting in JavaScript.

Hoisting is the engine moving declarations to the top of their scope during compile, before any code runs. Only declarations hoist — assignments stay in place.

// What you write            // How the engine sees it
console.log(x);              var x;
var x = 5;                   console.log(x); // undefined
                             x = 5;

sayHi();                     function sayHi() { ... }
function sayHi() {}          sayHi();

Rules

  • var: hoisted, initialized to undefined.
  • let/const: hoisted, not initialized — TDZ error if accessed early.
  • Function declarations: fully hoisted (body included).
  • Function expressions: only the variable hoists.
  • class declarations: in TDZ like let.
Question 12

What are Immediately Invoked Function Expressions (IIFE), and why are they used?

An IIFE is a function defined and called at the same time. Syntax: wrap in parens to make it an expression, then invoke.

(function() {
  const secret = "hidden";
  console.log("runs immediately");
})();

// Modern arrow form
(() => { /* ... */ })();

// With result capture
const config = (function() {
  const internal = loadStuff();
  return { api: internal.url };
})();

Why use one?

  • Scope isolation — pre-ES6 module workaround to avoid polluting global.
  • One-time setup — compute a value, discard locals.
  • UMD wrappers for libraries loaded via <script>.

ES6 modules and block-scoped let cover most of these use cases now, but IIFEs still appear in bundler output and library code.

Question 13

How do you handle errors in JavaScript using try-catch blocks?

try {
  const data = JSON.parse(input);
  risky(data);
} catch (err) {
  if (err instanceof SyntaxError) { /* bad JSON */ }
  else { /* rethrow or log */ throw err; }
} finally {
  cleanup();  // always runs — close handles, clear state
}

Key points

  • try/catch only catches synchronous throws in its block. Async errors need await + try/catch or Promise .catch.
  • finally runs even if try or catch returns.
  • Rethrow what you can't handle: catch (e) { log(e); throw e; }.
  • Narrow by type: check instanceof, don't string-match error messages.
  • Prefer custom error classes for domain failures so callers can switch on them.
Question 14

// Sync — blocks the thread
const data = readFileSync("big.csv");   // waits

// Async — schedules work, returns immediately
readFile("big.csv").then(process);      // loop keeps running
// or

In the browser, sync work blocks the UI (jank, unresponsive buttons). On a server, sync I/O blocks other requests. Async keeps the event loop free to handle other work while waiting.

Question 15

Explain the concept of promises and how they are used in JavaScript.

A Promise represents a value that will be available later — pending, fulfilled, or rejected. Once settled, it never changes.

const p = new Promise((resolve, reject) => {
  doThing(err => err ? reject(err) : resolve(42));
});

p.then(v => v * 2)          // chain — each .then returns a new Promise
 .then(v => console.log(v)) // 84
 .catch(err => console.error(err))
 .finally(() => cleanup());

// Combinators
Promise.all([a, b, c]);       // all resolved, or first rejection
Promise.allSettled([a, b]);   // all outcomes, no short-circuit
Promise.race([a, b]);         // first settled
Promise.any([a, b]);          // first fulfilled

The modern ergonomic form is async/await, which is syntactic sugar over promises — the underlying mechanics are identical.

Question 16

How does async/await syntax work in JavaScript?

async makes a function return a Promise. await pauses execution until a Promise settles, then unwraps its value (or throws on rejection).

async function loadUser(id) {
  const user = await fetch(`/user/${id}`).then(r => r.json());
  const orders = await fetch(`/orders?u=${user.id}`).then(r => r.json());
  return { user, orders };
}

loadUser(1).then(console.log).catch(console.error);

Under the hood

The engine transforms an async function into a generator-like state machine. Each await yields control; when the promise settles, the runtime resumes the function from that point.

⚠️
Sequential vs parallel: two sequential awaits run one after the other. For independent calls, fire first, await later: const [a, b] = await Promise.all([f1(), f2()]);
Question 17

What are higher-order functions, and can you give an example?

A higher-order function either takes a function as an argument, returns a function, or both.

// Takes a function
[1, 2, 3].map(x => x * 2);       // [2, 4, 6]
arr.filter(x => x > 0);
arr.reduce((a, b) => a + b, 0);

// Returns a function
function greet(greeting) {
  return name => `${greeting}, ${name}!`;
}
const hi = greet("Hi");
hi("Ibrahim");                    // "Hi, Ibrahim!"

// Both (decorator)
function withLogging(fn) {
  return function(...args) {
    console.log("calling", fn.name, args);
    return fn(...args);
  };
}

HOFs enable composition, abstraction over behavior (sorting, filtering), and patterns like currying and partial application.

Question 18

Explain the concept of currying in JavaScript.

Currying transforms f(a, b, c) into f(a)(b)(c) — each call takes one argument and returns a function expecting the next.

// Manual
const add = a => b => c => a + b + c;
add(1)(2)(3);  // 6

// Generic curry
function curry(fn) {
  return function curried(...args) {
    return args.length >= fn.length
      ? fn(...args)
      : (...more) => curried(...args, ...more);
  };
}
const log = curry((level, tag, msg) => `[${level}][${tag}] ${msg}`);
const warn = log("WARN");            // partial
const authWarn = warn("auth");       // further partial
authWarn("token expired");           // "[WARN][auth] token expired"

Useful for building reusable, configurable functions and for point-free programming in functional libraries (Ramda, lodash/fp).

Question 19

What is event delegation, and how does it improve performance?

Event delegation means attaching one listener on a common ancestor and using event.target to identify which descendant fired. It exploits event bubbling.

// Instead of 1000 listeners on 1000 rows:
document.querySelector("#list").addEventListener("click", e => {
  const row = e.target.closest(".row");
  if (row) handleRow(row.dataset.id);
});

Performance wins

  • One listener instead of N — less memory, faster setup.
  • Dynamic content works automatically — new rows don't need listeners attached.
  • Easier cleanup: remove one listener, not many.

Drawbacks: the event must bubble (some don't, like focus — use focusin). Need careful target checking to avoid false matches.

Question 20

How does JavaScript handle memory management and garbage collection?

JS uses automatic garbage collection — you don't free() anything. The engine periodically reclaims memory for unreachable objects.

Mark-and-sweep (the dominant algorithm)

  1. Start from roots (globals, current stack frames).
  2. Mark every object reachable by following references.
  3. Sweep away everything unmarked.

Modern engines (V8) also use generational GC — new objects live in a small "young" space collected often; survivors are promoted to an "old" space collected less frequently.

Common leak sources

  • Forgotten timers / intervals still referencing closures.
  • Detached DOM nodes kept alive by JS refs.
  • Global caches that grow without bound.
  • Event listeners not removed when components unmount.

Use WeakMap/WeakSet for caches keyed by object identity — the GC can collect keys that are otherwise unreferenced.

Question 21

What are the different ways to create an object in JavaScript?

// 1. Object literal
const obj = { a: 1 };

// 2. Constructor function
function Cat(name) { this.name = name; }
new Cat("Whiskers");

// 3. ES6 class
class Dog { constructor(name) { this.name = name; } }
new Dog("Rex");

// 4. Object.create — direct prototype
const proto = { greet() { return "hi"; } };
const instance = Object.create(proto);

// 5. Factory function
const makeUser = name => ({ name, greet: () => "hi" });

// 6. Object.assign / spread

const spread = { ...defaults, ...overrides };

// 7. Object.fromEntries — from key/value pairs

Prefer object literals / factories for data; classes for things with behavior and inheritance; Object.create when you need precise prototype control.

Question 22

How can you achieve inheritance in JavaScript using ES6 classes?

Use extends and super().

class Animal {
  constructor(name) { this.name = name; }
  speak() { return `${this.name} makes a sound.`; }
}

class Dog extends Animal {
  constructor(name, breed) {
    super(name);           // must call before using `this`
    this.breed = breed;
  }
  speak() {
    return super.speak() + " Woof!";   // access parent method
  }
  static create(name) { return new Dog(name, "unknown"); }
}

const d = new Dog("Rex", "lab");
d.speak();                 // "Rex makes a sound. Woof!"
d instanceof Animal;       // true
  • super(...) in the constructor calls the parent constructor (mandatory if you declare your own constructor in a subclass).
  • super.method() calls the parent's version of a method.
  • Static methods are inherited too — Dog.create works on subclasses.

Classes are sugar over prototype inheritance — Dog.prototype.__proto__ === Animal.prototype.

Question 23

Explain the difference between shallow copy and deep copy.

A shallow copy copies the top level; nested objects are shared (same reference). A deep copy recursively duplicates everything.

const orig = { a: 1, nested: { b: 2 } };

// Shallow
const s1 = { ...orig };
const s2 = Object.assign({}, orig);
s1.nested.b = 99;
orig.nested.b;           // 99  — shared!

// Deep
const d = structuredClone(orig);
d.nested.b = 42;
orig.nested.b;           // still 99 or whatever — isolated

Question 24

How do you debounce or throttle a function in JavaScript?

Both limit how often a function runs, but in opposite ways.

// Debounce — fires only after a quiet period
function debounce(fn, wait) {
  let t;
  return (...a) => {
    clearTimeout(t);
    t = setTimeout(() => fn(...a), wait);
  };
}

// Throttle — fires at most once per interval
function throttle(fn, wait) {
  let last = 0;
  return (...a) => {
    const now = Date.now();
    if (now - last >= wait) { last = now; fn(...a); }
  };
}
When to use
DebounceSearch-as-you-type, resize end, form validation
ThrottleScroll handlers, mousemove, rate-limited logging

Edge cases: should the first call fire immediately (leading edge)? The last queued one after quiet (trailing)? Libraries like lodash let you configure both.

How does the spread operator work, and what are its uses?

The spread operator (...) expands iterables or object properties in place.

Question 30

Explain the concept of tail call optimization in JavaScript.

Tail call optimization (TCO) eliminates the stack frame of a function whose last action is a call to another function, enabling deep/infinite recursion without stack overflow.

// Classic recursion — grows the stack
function factorial(n) {
  if (n <= 1) return 1;
  return n * factorial(n - 1);    // not a tail call — multiplication after
}

// Tail-recursive
function fact(n, acc = 1) {
  if (n <= 1) return acc;
  return fact(n - 1, n * acc);    // tail call — nothing after
}
⚠️
TCO is in the spec (ES2015) but in practice only Safari implements it. V8 (Chrome, Node) and Firefox do not, citing debugger/stack-trace trade-offs. Don't rely on it — use iteration or explicit trampolines for deep recursion in production.

Question 33

How do you handle deep cloning of objects containing circular references?

Circular references break naive recursion (JSON.stringify throws, simple recursion loops forever). Track visited objects in a WeakMap.

function deepClone(v, seen = new WeakMap()) {
  if (v === null || typeof v !== "object") return v;
  if (seen.has(v)) return seen.get(v);          // break cycle

  const out = Array.isArray(v) ? [] : {};
  seen.set(v, out);                             // register before recursing
  for (const key of Reflect.ownKeys(v)) {
    out[key] = deepClone(v[key], seen);
  }
  return out;
}

const a = { name: "A" };
a.self = a;                   // cycle
const c = deepClone(a);
c.self === c;                 // true — cycle preserved

Better: use structuredClone(a) — it handles cycles natively, plus Date, Map, Set, typed arrays, and more.

Question 34

Explain the concept of memoization and how it can be implemented in JavaScript.

Memoization caches a function's output by its inputs, trading memory for recomputation time.

function memoize(fn) {
  const cache = new Map();
  return function(...args) {
    const key = JSON.stringify(args);
    if (cache.has(key)) return cache.get(key);
    const result = fn.apply(this, args);
    cache.set(key, result);
    return result;
  };
}

const slowFib = n => n < 2 ? n : slowFib(n - 1) + slowFib(n - 2);
const fastFib = memoize(function f(n) {
  return n < 2 ? n : f(n - 1) + f(n - 2);
});
fastFib(50);   // instant; slowFib(50) takes forever

Caveats

  • Only for pure functions (same input → same output).
  • JSON.stringify keys fail for objects with cycles and order-sensitive content.
  • Cache grows unbounded — for long-running processes, use an LRU cache.
  • For object-keyed caches, use WeakMap so entries GC with the key.

Question 37

What are generators, and how do they differ from regular functions?

Generators are functions declared with function* that can pause (yield) and resume. They return an iterator.

function* counter() {
  let i = 0;
  while (true) yield i++;
}
const c = counter();
c.next();   // { value: 0, done: false }
c.next();   // { value: 1, done: false }

}
const e = echo();
e.next();      // { value: "first" }
e.next("hi");  // { value: "got hi" }

Question 38

How does the yield keyword work in JavaScript?

yield pauses a generator and surfaces a value to the caller. The generator resumes when .next(v) is called again, with v becoming the result of the yield expression.

function* g() {
  const a = yield 1;           // pause, surface 1
  const b = yield a + 10;      // resume with caller's value
  return a + b;
}
const it = g();
it.next();       // { value: 1, done: false }
it.next(5);      // { value: 15, done: false }   (a=5)
it.next(7);      // { value: 12, done: true }    (b=7, returns 5+7)

// yield* delegates to another iterable
function* inner() { yield 1; yield 2; }
function* outer() {
  yield 0;
  yield* inner();   // flattens
  yield 3;
}
[...outer()];    // [0, 1, 2, 3]

Outside a generator body, yield is a syntax error. In async generators, yield pairs with for await...of.

Question 40

What is the Reflect API, and how is it used?

Reflect is a built-in namespace mirroring Proxy's trap operations as regular functions. It provides a cleaner API for programmatic object manipulation.

// Old way                      // Reflect way

"key" in obj;                    Reflect.has(obj, "key");
Object.keys(obj);                Reflect.ownKeys(obj);    // includes symbols
new Cls(...args);                Reflect.construct(Cls, args);
fn.apply(ctx, args);             Reflect.apply(fn, ctx, args);

// Forwarding in a Proxy handler
const p = new Proxy(target, {
  get(t, prop, recv) {
    console.log("get", prop);
    return Reflect.get(t, prop, recv);   // default behavior

    return Reflect.set(t, prop, val, recv);

});

Key wins: Reflect returns booleans instead of throwing (friendlier error handling), and provides a matched default for every Proxy trap so you can layer logic while preserving semantics.

How do you perform method chaining in JavaScript?

What are the potential issues with floating-point arithmetic in JavaScript?

JS numbers are IEEE-754 doubles — not all decimals can be represented exactly.

Question 43

How do you implement a custom iterator in JavaScript?

Any object is iterable if it implements Symbol.iterator, returning an object with a next() method that yields { value, done }.

class Range {
  constructor(start, end) { this.start = start; this.end = end; }
  [Symbol.iterator]() {
    let current = this.start;
    const end = this.end;
    return {
      next() {
        return current < end
          ? { value: current++, done: false }
          : { value: undefined, done: true };
      },
    };
  }
}

for (const n of new Range(1, 4)) console.log(n);  // 1, 2, 3
[...new Range(1, 4)];    // [1, 2, 3]

// Easier with a generator
class Range2 {
  *[Symbol.iterator]() {
    for (let i = this.start; i < this.end; i++) yield i;
  }
}

Any iterable works with for...of, spread, destructuring, Array.from, Map/Set constructors, and Promise.all.

A monad is a pattern for chaining computations inside a wrapper (container) while abstracting boilerplate like null-checks or error handling. It needs two operations: of (wrap a value) and flatMap/chain (apply a function that returns a new wrapped value, and avoid double-wrapping).

// Maybe monad — skip the rest of the chain if null/undefined
class Maybe {
  constructor(value) { this.value = value; }
  static of(v) { return new Maybe(v); }
  map(fn)     { return this.value == null ? this : Maybe.of(fn(this.value)); }
  flatMap(fn) { return this.value == null ? this : fn(this.value); }
}

Maybe.of(user)
  .map(u => u.address)
  .map(a => a.city)
  .map(c => c.toUpperCase())
  .value;     // safely navigates or yields undefined

// You've used monads already:
// - Promise is a monad (then = flatMap, resolve = of)
// - Array is a monad (flatMap, of = Array.of)

The abstraction is more popular in FP languages. In JS, optional chaining (?.) covers the Maybe case and Promises cover async.

Question 64

Explain the concept of function composition.

Function composition chains functions so the output of one feeds the input of the next. Builds complex transformations from small pure pieces.

const compose = (...fns) => x => fns.reduceRight((v, f) => f(v), x);
const pipe    = (...fns) => x => fns.reduce((v, f) => f(v), x);

const addOne = x => x + 1;
const double = x => x * 2;
const square = x => x * x;

const f = compose(square, double, addOne);  // square(double(addOne(x)))
f(3);    // addOne(3)=4, double=8, square=64

const g = pipe(addOne, double, square);     // same result, left-to-right reading
g(3);    // 64

Pipe (left-to-right) reads like Unix pipes. Compose (right-to-left) matches mathematical notation f∘g. Pick one convention per codebase.

Works well with curried, unary functions — this style is idiomatic in Ramda and lodash/fp.

Question 69

How do you debounce a function in JavaScript?

Debouncing delays execution until a burst of calls settles. See also Q52 (Basic). This question appears in the Tricky set too — the gotcha here is preserving this and arguments:

function debounce(fn, wait) {
  let t;
  return function(...args) {        // regular function — receives caller's `this`
    clearTimeout(t);
    t = setTimeout(() => fn.apply(this, args), wait);
  };
}

// Advanced: leading + trailing edges, cancel, flush
function debounceAdv(fn, wait, { leading = false } = {}) {
  let t, pendingArgs, pendingThis;
  const run = () => { fn.apply(pendingThis, pendingArgs); t = null; };
  const wrapped = function(...args) {
    pendingArgs = args; pendingThis = this;
    if (!t && leading) fn.apply(this, args);
    clearTimeout(t);
    t = setTimeout(run, wait);
  };
  wrapped.cancel = () => { clearTimeout(t); t = null; };
  wrapped.flush  = () => { if (t) { clearTimeout(t); run(); } };
  return wrapped;
}

Question 72

How do you handle circular dependencies in JavaScript modules?

Circular dependencies (A imports B, B imports A) usually indicate a design problem but sometimes happen in practice. CommonJS and ES modules handle them differently.

ES modules

Imports are live bindings. At module evaluation, you see the partially-initialized exports. Functions are fine (they run later); top-level access to an unevaluated export gives undefined or a TDZ error.

// a.js
import { b } from "./b.js";
export const a = 1;
export function useB() { return b; }   // safe — called later

// b.js
import { a } from "./a.js";
export const b = 2;
export function useA() { return a; }

Strategies

  • Extract shared code into a third module both can import.
  • Lazy-require inside functions (CommonJS): function() { const b = require("./b"); }.
  • Dependency injection — pass the dependency in instead of importing it.
  • Invert control — make one module a consumer of events the other emits.

Tools like madge or ESLint's import/no-cycle detect cycles at lint time.

Question 77

How do you implement a linked list in JavaScript?

class LinkedList {
  constructor() { this.head = null; this.size = 0; }

  push(value) {
    const node = { value, next: null };
    if (!this.head) { this.head = node; }
    else {
      let cur = this.head;
      while (cur.next) cur = cur.next;
      cur.next = node;
    }
    this.size++;
  }

  prepend(value) {
    this.head = { value, next: this.head };
    this.size++;
  }

  remove(value) {
    if (!this.head) return false;
    if (this.head.value === value) {
      this.head = this.head.next; this.size--; return true;
    }
    let cur = this.head;
    while (cur.next && cur.next.value !== value) cur = cur.next;
    if (cur.next) { cur.next = cur.next.next; this.size--; return true; }
    return false;
  }

  *[Symbol.iterator]() {
    for (let cur = this.head; cur; cur = cur.next) yield cur.value;
  }
}

Advantages: O(1) prepend, no resizing. Drawbacks: O(n) random access, poor cache locality. Doubly-linked lists add a prev pointer for O(1) removal when you already have the node.

Question 78

Explain the concept of a binary tree and how to implement it in JavaScript.

A binary tree is a hierarchy where each node has up to two children (left, right). A BST orders values so left < node < right.

class BST {
  constructor() { this.root = null; }

  insert(value) {
    const node = { value, left: null, right: null };
    if (!this.root) { this.root = node; return; }
    let cur = this.root;
    while (true) {
      const side = value < cur.value ? "left" : "right";
      if (!cur[side]) { cur[side] = node; return; }
      cur = cur[side];
    }
  }

  has(value) {
    let cur = this.root;
    while (cur) {
      if (value === cur.value) return true;
      cur = value < cur.value ? cur.left : cur.right;
    }
    return false;
  }

  *inorder(node = this.root) {
    if (!node) return;
    yield* this.inorder(node.left);
    yield node.value;
    yield* this.inorder(node.right);
  }
}

const tree = new BST();
[5, 3, 7, 1, 4].forEach(v => tree.insert(v));
[...tree.inorder()];   // [1, 3, 4, 5, 7]

Unbalanced BSTs degrade to O(n). Self-balancing variants (Red-Black, AVL) keep operations at O(log n).

"Empty" is context-dependent. An empty Map/Set has no enumerable keys but .size === 0. For general containers, check each type's appropriate emptiness signal.

Question 84

What are the potential security issues in JavaScript, and how can they be mitigated?

Common attack surfaces

  • XSS (Cross-Site Scripting): injected script runs in your origin. Mitigate with strict escaping (textContent not innerHTML), templating that escapes by default (React, Vue), and a Content Security Policy header.
  • CSRF: third-party site triggers authenticated requests. Use SameSite cookies, CSRF tokens, check Origin/Referer.
  • Clickjacking: your page framed to capture clicks. Send X-Frame-Options: DENY or frame-ancestors CSP.
  • Prototype pollution: untrusted input writes to __proto__, poisoning all objects. Validate keys, use Map or Object.create(null).
  • Injection via eval/Function: never pass untrusted strings to eval, new Function, or setTimeout(string).
  • Supply chain: a compromised npm dep runs your users' code. Audit with npm audit, pin versions, use lockfiles, consider --ignore-scripts.
  • Leaking secrets: API keys in client bundles are public. Proxy through your server.
  • Open redirects: validate URLs against an allowlist.

HTTPS everywhere; Secure, HttpOnly, SameSite cookies for auth; a strict CSP; and input validation at trust boundaries cover the majority of practical threats.

Question 85

How do you implement a stack and a queue in JavaScript?

// Stack (LIFO) — use array push/pop
class Stack {
  #data = [];
  push(v) { this.#data.push(v); }
  pop()   { return this.#data.pop(); }
  peek()  { return this.#data[this.#data.length - 1]; }
  get size() { return this.#data.length; }
  get isEmpty() { return this.#data.length === 0; }
}

// Queue (FIFO) — array.shift is O(n); use a linked list or two-stack
class Queue {
  #head = null; #tail = null; #size = 0;
  enqueue(v) {
    const node = { value: v, next: null };
    if (this.#tail) this.#tail.next = node; else this.#head = node;
    this.#tail = node; this.#size++;
  }
  dequeue() {
    if (!this.#head) return undefined;
    const v = this.#head.value;
    this.#head = this.#head.next;
    if (!this.#head) this.#tail = null;
    this.#size--;
    return v;
  }
  get size() { return this.#size; }
}

Arrays give O(1) stack ops. Naive queue with arr.shift() is O(n); a linked list or two-stack queue keeps enqueue/dequeue at O(1).

Question 88

What are decorators in JavaScript, and how are they used?

Decorators wrap classes, methods, fields, or accessors to add behavior at declaration time. Stage 3 TC39 proposal; widely used in TypeScript and frameworks.

// Method decorator — log every call

  return function(...args) {
    console.log(`${name}(${args})`);
    return original.call(this, ...args);
  };
}

class Api {
  @log
  fetch(url) { return fetch(url); }
}

// Class decorator
function registered(Cls) {
  registry.set(Cls.name, Cls);
  return Cls;
}

@registered
class Widget {}

Uses

  • Framework metadata (@Component, @Injectable in Angular).
  • Validation (@MinLength(5)).
  • Logging, memoization, access control.
  • Route registration in backend frameworks.

Question 90

What is the purpose of the async keyword?

async on a function does two things: makes it return a Promise, and allows await inside it.

async function f() { return 42; }
f();          // Promise<42>, not 42

async function g() { throw new Error("fail"); }
g();          // rejected Promise — never an uncaught exception from the call

// Compared to a regular function returning a promise — identical externally
function f2() { return Promise.resolve(42); }   // same observable behavior as async f

// But only async functions can use await
async function h() {
  const x = await fetch("/api");
  return x.json();
}

Top-level await (ES2022)

In ES modules, await works at the module's top level without wrapping in async:

// my-module.js
const config = await fetch("/config.json").then(r => r.json());
export default config;

Consumers importing this module wait for its evaluation to finish. Use sparingly — blocks dependents' evaluation.

Question 91

How do you implement a custom map function?

Array.prototype.myMap = function(fn, thisArg) {
  if (this == null) throw new TypeError();
  if (typeof fn !== "function") throw new TypeError();
  const arr = Object(this);
  const len = arr.length >>> 0;
  const out = new Array(len);
  for (let i = 0; i < len; i++) {
    if (i in arr) out[i] = fn.call(thisArg, arr[i], i, arr);
  }
  return out;
};

[1, 2, 3].myMap(x => x * 2);   // [2, 4, 6]

Details the real spec gets right

  • Non-array this (like {length: 3, 0: "a", 2: "b"}) is coerced to an object and length to uint32.
  • Sparse arrays: skip holes (i in arr check), don't call fn on them.
  • Three callback args: (element, index, array).
  • Second argument to map: thisArg.
  • Returns a new array, doesn't mutate the original.

Writing your own helps internalize the full behavior — which is more nuanced than arr.forEach(...); return result.

How do you handle timeouts with promises?

function withTimeout(promise, ms, message = "timeout") {
  let timer;
  const timeout = new Promise((_, reject) => {
    timer = setTimeout(() => reject(new Error(message)), ms);
  });
  return Promise.race([promise, timeout])
    .finally(() => clearTimeout(timer));
}

}

Question 95

How do you detect memory leaks in a JavaScript application?

Signs of a leak

  • Steady upward trend in memory after sustained use.
  • Performance degrades over time, eventually crashes or swap-thrashes.
  • Heap snapshots show growing retained size for the same objects.

Tools

  • Chrome DevTools → Memory tab: Heap snapshots (compare two to find growth), Allocation instrumentation on timeline, Allocation sampling.
  • performance.memory — quick totalJSHeapSize sanity check.
  • Node: --inspect + Chrome DevTools, or process.memoryUsage().
  • clinic.js, 0x for production profiling.

Common culprits

  • Event listeners not removed on component unmount.
  • Timers/intervals still referencing closures.
  • Detached DOM nodes held by JS variables.
  • Module-level caches growing without bound.
  • Unresolved promises accumulating (never resolve/reject).

Workflow

Take snapshot → interact with the suspected area → take snapshot → diff. Filter to "Objects allocated between snapshots" that are still retained. Follow retaining paths to find the root reference.

Question 96

What are the benefits of using immutability in JavaScript?

  • Predictability: state snapshots are safe to share, cache, and pass around without defensive copies.
  • Cheap equality checks: reference equality implies data equality — enables memoization, React's memo/useMemo, Redux's selectors.
  • Time-travel debugging: keep every past state; undo/redo free.
  • Concurrency safety: no shared-mutable-state bugs (less relevant in single-threaded JS, more for workers).
  • Easier reasoning: functions that don't mutate inputs are easier to test and refactor.
  • Framework alignment: React, Redux, and modern state libraries assume immutability for change detection.

Costs

  • GC pressure — more allocations.
  • Boilerplate if done by hand; libraries like Immer hide this.
  • Can be slower for large, deep updates without structural sharing (immutable.js, Immer).

Net win for application state; pragmatic mutation still wins in tight algorithmic code.

Question 99

How do you implement a binary search algorithm in JavaScript?

function binarySearch(arr, target) {
  let lo = 0, hi = arr.length - 1;
  while (lo <= hi) {
    const mid = (lo + hi) >>> 1;        // avoids overflow, floors
    if (arr[mid] === target) return mid;
    if (arr[mid] < target) lo = mid + 1;
    else hi = mid - 1;
  }
  return -1;
}

binarySearch([1, 3, 5, 7, 9, 11], 7);   // 3
binarySearch([1, 3, 5, 7, 9, 11], 6);   // -1

// Recursive (less efficient, clearer)
function bs(arr, target, lo = 0, hi = arr.length - 1) {
  if (lo > hi) return -1;
  const mid = (lo + hi) >>> 1;
  if (arr[mid] === target) return mid;
  return arr[mid] < target
    ? bs(arr, target, mid + 1, hi)
    : bs(arr, target, lo, mid - 1);
}

// Find insertion index (lower bound)
function insertIdx(arr, target) {
  let lo = 0, hi = arr.length;
  while (lo < hi) {
    const mid = (lo + hi) >>> 1;
    if (arr[mid] < target) lo = mid + 1; else hi = mid;
  }
  return lo;
}

O(log n) time, O(1) space (iterative). Requires sorted input — sort cost is O(n log n), so binary search pays off with multiple lookups.

Question 100

Explain the concept of the BigInt type and its usage.

BigInt is a numeric primitive for arbitrary-precision integers, for values beyond Number.MAX_SAFE_INTEGER (2⁵³ − 1).

// Literal — n suffix
const big = 9007199254740993n;
typeof big;           // "bigint"

// Constructor — from number or string
BigInt(100);                    // 100n
BigInt("9007199254740993");     // 9007199254740993n
BigInt(1.5);                    // RangeError — non-integer

// Arithmetic
big + 1n;             // 9007199254740994n
big * 2n;             // doubles
big / 2n;             // integer division — truncates

// Mixing with Number throws
1n + 1;               // TypeError — must be same type
Number(1n) + 1;       // 2
1n + BigInt(1);       // 2n

// Comparisons work across types
1n === 1;             // false (types differ)
1n == 1;              // true  (coercion)
1n < 2;               // true

Caveats

  • Cannot be mixed with Number in arithmetic — convert explicitly.
  • Not JSON-serializable (must stringify manually).
  • No fractional values — use a decimal library for currency math.

Use for: cryptography, huge IDs (e.g., Twitter snowflake), scientific integer math, file sizes > 2⁵³ bytes.
