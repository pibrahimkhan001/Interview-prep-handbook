## 🔀 Data Flow

```
 Excel                                       JSON                       HTML pages
 ─────                                       ────                       ──────────
 Interview_Prep.xlsx
       │
       │  extract_questions.py
       ▼
 data/questions.json  ──────────────────┐
                                        │
 content/{topic}_{level}.py  ───────────┼─▶  build_chapters.py  ──▶  chapters/*.html
 (hand-written ANSWERS dict)            │
                                        │
                                        └─▶  build_index.py      ──▶  index.html
```

- `questions.json` is the machine-readable truth for what questions exist.
- `content/*.py` modules provide the detailed HTML answers.
- Chapters without a matching content module render as **stubs** — question list + "coming soon" note.

## 📦 Content Module Format

Every content module is a Python file exporting a single `ANSWERS` dict.

```python
# content/python_basic.py
"""Python · Basic · Detailed answers. Each value is an HTML snippet."""

ANSWERS = {}

ANSWERS[1] = r'''
<p>Python is a high-level, interpreted, general-purpose programming language...</p>
<pre><code>print("Hello, World!")</code></pre>
'''

ANSWERS[2] = r'''
<p>...</p>
'''
```

### Rules

1. **Filename format:** `{topic_slug}_{level_slug}.py` — lowercase, underscores (not hyphens), no spaces. E.g., `javascript_advanced_coding.py`, `ci_cd_pipeline_basic.py`.
2. **Key type:** 1-indexed integers matching question numbers in `questions.json`.
3. **Value type:** raw HTML string (use `r'''...'''` to avoid escape headaches).
4. **Completeness:** if you provide a module, you should provide all N answers. Partial modules render answered questions with detail and missing ones with a placeholder.
5. **No imports** from within content modules — they're loaded via `importlib`.

### Slug derivation

The build script computes the module filename from topic + level:

| Topic | Level | Slug |
| --- | --- | --- |
| JavaScript | Basic | `javascript_basic` |
| JavaScript | Advanced Coding | `javascript_advanced_coding` |
| Node.Js | Scenario Based | `nodejs_scenario` |
| CI/CD Pipeline | Basic | `cicd_pipeline_basic` |
| System Design MERN Stack | Advanced | `system_design_mern_advanced` |

Rule of thumb: lowercase everything, strip punctuation, replace spaces with underscores, collapse repeats.

## 🛠️ Build Pipeline

### Regenerate everything

```bash
cd interview-prep-guide
python3 scripts/build_chapters.py
python3 scripts/build_index.py
```

Takes ~1 second. No dependencies beyond Python 3.8+.

### What each script does

**`scripts/extract_questions.py`** — One-time Excel parser. Reads `/mnt/project/Ibrahim_Interview_Prep.xlsx`, extracts the "Interview Practice Questions" sheet, and writes `data/questions.json`. Only re-run if the source Excel changes.

**`scripts/site_data.py`** — Pure data module. Defines the Parts taxonomy (which topics live in which section) and provides slug/title helpers. Imported by the two build scripts.

**`scripts/html_common.py`** — Shared HTML fragments. Header bar, brand mark, full-site sidebar, footer, `<head>` boilerplate. Keeps chapter and index pages consistent.

**`scripts/build_chapters.py`** — For every (topic, level) in `questions.json`:
1. Look for `content/{slug}.py`. If present, import it and read the `ANSWERS` dict.
2. Render the chapter page: shell (head, header, sidebar) + chapter header + numbered Q&A blocks.
3. Write to `chapters/{slug}.html`.
4. Print status line: `[detailed]` if `ANSWERS` loaded, `[stub]` otherwise.

**`scripts/build_index.py`** — Renders the homepage: hero, roadmap by part, chapter grid with counts, footer. Writes `index.html`.

## ➕ How to Add Content for a New Topic/Level

1. **Pick the target:** e.g., Python Basic.
2. **Check the question list:**
   ```bash
   python3 -c "import json; d=json.load(open('data/questions.json'));
   print('\n'.join(f'{i}. {q}' for i, q in enumerate(d['data']['Python']['Basic'], 1)))"
   ```
3. **Create the module:** `content/python_basic.py` with an `ANSWERS = {}` dict.
4. **Write answers one at a time** (or in batches):
   ```python
   ANSWERS[1] = r'''<p>...</p>'''
   ```
   Follow `STYLE_GUIDE.md` for depth and conventions.
5. **Verify periodically:**
   ```bash
   python3 -c "import content.python_basic as m;
   print(len(m.ANSWERS), sorted(m.ANSWERS.keys()) == list(range(1, 101)))"
   ```
6. **Rebuild:**
   ```bash
   python3 scripts/build_chapters.py
   ```
7. **Check output size** — populated chapters are 100–200 KB. Anything under 80 KB means thin content.
8. **Update `PROJECT_STATE.md`** — mark the level ✅.
9. **Log in `CHANGELOG.md`** — add a session entry.

## 🧩 Extending the Site

### Adding a new topic (not in source Excel)

1. Add a new key to `scripts/site_data.py` `TOPIC_META` (title, lede, read-time estimate per level).
2. Add it to a Part in `PARTS` (or create a new Part).
3. Add questions to `data/questions.json` under a new top-level topic key.
4. Run `build_chapters.py` + `build_index.py`.

### Adding a new level (e.g., "Interview Edge Cases")

1. Add to `LEVELS_BY_TOPIC` in `scripts/site_data.py` for each topic that has this level.
2. Add questions in `data/questions.json`.
3. Create `content/{topic}_{slug}.py` modules.
4. Rebuild.

### Changing the theme

- Colors and spacing: edit CSS custom properties at the top of `assets/styles.css`.
- Typography: swap Google Fonts link in `scripts/html_common.py::SHELL_HEAD`.
- Layout: `.layout`, `.sidebar`, `.main` classes in `styles.css`.
- Dark mode: `:root[data-theme="dark"]` block.

## 🔐 Invariants

Things that should always hold:

- **Chapter count = 49** (matches the level count across all topics in `questions.json`).
- **Every chapter has a valid `<link rel="stylesheet" href="../assets/styles.css">`** — relative path from `chapters/` to `assets/`.
- **No absolute file:// paths** anywhere — the site must work when dropped in any directory.
- **No external runtime dependencies** — Google Fonts loads from CDN (cached) but nothing else is required. Offline = fonts fall back to system fonts.
- **Python 3.8+ only** for build scripts — no third-party deps.
